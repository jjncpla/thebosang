# TBSS 사건별 첨부파일 기능 구현 태스크

## 개요
재해자의 사건(Case)별로 수발신 문서를 첨부파일 형태로 저장/다운로드하는 기능.
저장소: Railway PostgreSQL (Prisma Bytes 타입) — 추가 인프라 불필요.
UI 위치: 재해자 상세 페이지 좌측 패널, "비고" 입력란 아래.

---

## 1단계: Prisma 스키마 추가

### prisma/schema.prisma에 CaseAttachment 모델 추가

```prisma
model CaseAttachment {
  id            String   @id @default(cuid())
  caseId        String
  case          Case     @relation(fields: [caseId], references: [id], onDelete: Cascade)
  fileName      String          // 원본 파일명 (예: "장해급여결정통지서.pdf")
  fileSize      Int             // 바이트 단위
  mimeType      String          // "application/pdf", "image/jpeg" 등
  fileData      Bytes           // 실제 파일 바이너리
  category      String?         // 분류: SENT(발신) / RECEIVED(수신) / INTERNAL(내부) / OTHER(기타)
  description   String?         // 파일 설명 (선택)
  uploadedById  String?         // 업로드한 사용자 ID
  uploadedBy    User?    @relation(fields: [uploadedById], references: [id])
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@index([caseId])
  @@index([caseId, category])
}
```

### Case 모델에 relation 추가
```prisma
// Case 모델 내부에 추가:
attachments   CaseAttachment[]
```

### User 모델에 relation 추가 (auth.prisma 또는 schema.prisma의 User 모델)
```prisma
// User 모델 내부에 추가:
uploadedAttachments CaseAttachment[]
```

### 주의사항
- FK 삭제 순서에 CaseAttachment 추가: CaseAttachment → (기존 순서) → Case → Patient
- `onDelete: Cascade`로 Case 삭제 시 첨부파일도 함께 삭제
- 스키마 변경 후 Railway 배포: 1회용 init API 또는 로컬 `prisma db push` (외부 프록시 URL)

## 2단계: API 라우트 구현

### 2-1. 파일 업로드 API
**경로**: `app/api/cases/[caseId]/attachments/route.ts`

```typescript
// POST: 파일 업로드 (multipart/form-data)
// - request.formData()로 파일 수신
// - 파일 크기 제한: 10MB
// - 허용 MIME: application/pdf, image/jpeg, image/png, image/gif, 
//   application/msword, application/vnd.openxmlformats-officedocument.wordprocessingml.document,
//   application/vnd.ms-excel, application/vnd.openxmlformats-officedocument.spreadsheetml.sheet,
//   application/hwp, application/x-hwp
// - category, description은 formData에서 함께 수신
// - Buffer.from(await file.arrayBuffer())로 변환 후 Prisma Bytes로 저장
// - 세션 체크: 로그인 사용자만 업로드 가능
// - 응답: { id, fileName, fileSize, mimeType, category, description, createdAt }
//   (fileData는 응답에 포함하지 않음)

// GET: 해당 Case의 첨부파일 목록 조회
// - fileData 제외하고 메타데이터만 반환
// - category별 필터링 지원 (?category=SENT)
// - createdAt DESC 정렬
```

### 2-2. 파일 다운로드/삭제 API
**경로**: `app/api/cases/[caseId]/attachments/[attachmentId]/route.ts`

```typescript
// GET: 파일 다운로드
// - fileData를 Buffer로 변환
// - Content-Disposition: attachment; filename="원본파일명"
// - Content-Type: 저장된 mimeType
// - 한글 파일명 인코딩: encodeURIComponent 또는 RFC 5987

// DELETE: 파일 삭제
// - 세션 체크: 업로드한 본인 또는 ADMIN/SENIOR_MANAGER만 삭제 가능
// - 응답: { success: true }
```

### 2-3. 파일 미리보기 API (선택)
**경로**: `app/api/cases/[caseId]/attachments/[attachmentId]/preview/route.ts`

```typescript
// GET: 인라인 표시 (이미지/PDF)
// - Content-Disposition: inline
// - 이미지: 직접 표시
// - PDF: 브라우저 내장 뷰어로 표시
```

## 3단계: UI 컴포넌트 구현

### 3-1. CaseAttachments 컴포넌트
**경로**: `app/components/CaseAttachments.tsx`

#### Props
```typescript
interface CaseAttachmentsProps {
  caseId: string;
}
```

#### 기능 요구사항
1. **드래그 앤 드롭 영역**
   - 점선 테두리 박스, "파일을 드래그하거나 클릭하여 첨부" 안내 텍스트
   - 드래그 오버 시 시각적 피드백 (배경색 변경, 테두리 색 변경)
   - 클릭 시 파일 선택 다이얼로그 (hidden input[type=file])
   - 다중 파일 선택 지원
   - 10MB 초과 파일 alert

2. **파일 목록 표시**
   - 파일명, 크기(KB/MB 자동변환), 분류, 업로드 일시 표시
   - 파일 아이콘: PDF/이미지/엑셀/한글 등 확장자별 다른 아이콘 (이모지 또는 SVG)
   - 다운로드 버튼 (클릭 시 브라우저 다운로드)
   - 삭제 버튼 (확인 모달 후 삭제, 권한 체크)
   - 이미지/PDF는 미리보기 링크 제공

3. **분류 선택**
   - 업로드 시 카테고리 선택 드롭다운: 발신 / 수신 / 내부자료 / 기타
   - 파일 설명 입력 (선택, 한 줄 텍스트)
   - 목록에서 카테고리별 필터 탭 (전체/발신/수신/내부/기타)

4. **업로드 상태 표시**
   - 업로드 중 프로그레스 바 또는 스피너
   - 업로드 완료/실패 토스트 메시지

#### 스타일 가이드 (기존 TBSS 스타일 유지)
- 브랜드 컬러: 스카이블루 #29ABE2, 라임그린 #8DC63F, 딥그린 #006838
- 기존 인라인 스타일/Tailwind 혼재 스타일 따르기
- 비고 입력란과 동일한 width, 좌측 패널 내부에 자연스럽게 배치
- 섹션 제목: "첨부파일" (비고와 동일한 스타일의 레이블)

### 3-2. 재해자 상세 페이지에 컴포넌트 삽입

**대상 파일**: 재해자 상세 페이지 (patients/[id] 관련 컴포넌트)
- 좌측 패널에서 "비고" 입력란 바로 아래에 <CaseAttachments caseId={selectedCaseId} /> 삽입
- 현재 선택된 사건(Case)의 ID를 전달
- 사건이 선택되지 않은 경우(소음성 난청 탭 미선택) 비활성 또는 숨김

**주의**: 수술적 수정 원칙 — 기존 비고 영역, 재해자 정보 영역, 우측 사건 상세 영역 일체 건드리지 않음.
비고 textarea 다음 블록에 CaseAttachments 컴포넌트만 추가.

## 구현 순서 (Claude Code 실행 순서)

1. prisma/schema.prisma에 CaseAttachment 모델 추가 + Case/User relation 추가
2. npx prisma generate (로컬 타입 생성)
3. API 라우트 생성: upload/list + download/delete
4. CaseAttachments.tsx 컴포넌트 생성
5. 재해자 상세 페이지에 컴포넌트 삽입 (비고 아래)
6. 로컬 테스트
7. git push → Railway 배포
8. 스키마 마이그레이션 (init API 또는 로컬 push)
9. 프로덕션 테스트

## 파일 크기/용량 예측

- 현재 820건 × 건당 평균 5~10개 파일 × 평균 2MB = 약 8~16GB
- Railway PostgreSQL 무료 범위 초과 시 → Cloudflare R2로 마이그레이션 고려
- fileData 컬럼만 R2로 이전하고 메타데이터는 PostgreSQL 유지하는 구조로 설계해둠
- 마이그레이션 시 CaseAttachment에 `storageType` (DB/R2) + `externalUrl` 필드 추가만 하면 됨
