그 다음 아래 항목들을 중점적으로 진단해줘.

## 진단 항목

### 1. 아키텍처 구조 검토
- Next.js App Router 컨벤션에서 벗어난 파일 배치가 있는지
- Server Component / Client Component 경계가 잘못 설정된 부분이 있는지
- API Route 설계가 REST 원칙 또는 기존 패턴에서 이탈한 부분이 있는지

### 2. Prisma 스키마 / DB 안정성
- relation이 깨지거나 cascade 동작이 의도와 다를 수 있는 부분
- 추후 마이그레이션이 어려워질 수 있는 구조 (nullable 남발, FK 누락 등)
- B방식 FK연결 원칙(Case–ObjectionReview–ObjectionCase)과 충돌 여부

### 3. 장기 개발 장애물
- 지금은 돌아가지만 데이터가 쌓였을 때 문제가 생길 수 있는 쿼리 또는 구조
- 기능 확장 시 이 부분을 뜯어야 하는 상황이 생길 수 있는 패턴
- 수술적 수정 원칙을 위반해서 파급 범위가 넓어질 수 있는 부분

### 4. 잠재적 충돌
- 변경된 파일이 건드리지 않은 다른 모듈에 영향을 줄 수 있는지
- auth 권한 체계(ADMIN/조직관리자/STAFF/이산계정)와 새 기능의 정합성
- import 경로 또는 타입 정의의 불일치
