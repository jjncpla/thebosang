## 3. 조직 구조

### 더보상 조직
- 본사 + 전국 지사 (울산, 부산경남, 서울북부, 경기안산, 전북익산, 경북구미, 경기의정부, 강원동해, 전남여수, 대구, 부산중부, 경기수원 등)
- 직군: 노무사 / 내근직 / 외근직

### TF 개념
- TF = "사건군" (임시 조직 아님, 영업 경로별 사건 묶음)
- **더보상TF**: 더보상 소속이 영업한 사건 (모든 상병)
- **이산TF**: 노무법인 이산 직원이 영업한 사건 (소음성 난청 전용)
- 지사별 담당 TF 고정 (배정표 별도 전달 예정)

### 노무법인 이산
- 협력 법인. 이산 직원 영업 → 더보상 수행
- TBSS 접근: 사건 조회 페이지만 허용

## 7. 핵심 아키텍처 원칙

1. **Patient → Case → Detail** 계층 구조 유지 (재해자 1명 = 복수 사건 가능)
2. **B방식 FK**: `Case → ObjectionReview → ObjectionCase` 명시적 caseId FK
3. **수술적 수정**: 기존 파일 전체 재작성 금지. 필요한 부분만 추가/교체
4. **클라이언트 사이드 Excel 파싱**: 서버 OOM 방지 (BATCH_SIZE=50 JSON 배치)
5. **긴 지시사항은 task.txt**: Claude Code CLI 터미널 붙여넣기 한계 대응
6. **Claude 일원화**: ChatGPT 병행 금지 (맥락 충돌)
