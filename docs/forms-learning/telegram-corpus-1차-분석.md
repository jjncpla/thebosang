## A. 진찰요구서 방 — 메시지 템플릿 (정형도 99%)

### 패턴
```
{재해자명} 님 {상병명}{진찰종류} 진찰요구서 전달드립니다!
[{TF명} / {담당자}]

원스톱 : {병원명}
```

- "`{이름} 님 ... 진찰요구서`" 매치율: **99/100**
- 상병의 95%가 **난청** (이 방은 사실상 *난청 진찰요구서 전용 방*)
- TF/담당자 표기 `[TF명 / 담당자]` 매치율 52% — 표기 흔들림 있음 (`(TF명)` / `[TF명-담당자]` 등 변종)

### 정규식 후보
```python
PAT_요구서 = re.compile(
    r'(?P<name>\S{2,4})\s*님\s*'
    r'(?P<disease>난청|진폐|COPD|폐질환|근골[가-힣]*)?\s*'
    r'(?P<kind>특별진찰|재특진|전문조사|진찰)?\s*'
    r'(?:및\s*전문조사)?\s*진찰요구서\s*전달드립니다',
    re.UNICODE,
)

PAT_TF담당 = re.compile(r'[\[\(]\s*(?P<tf>[가-힣A-Za-z0-9]+TF)\s*[/\-]\s*(?P<staff>[가-힣 ]+?)\s*[\]\)]')
PAT_원스톱 = re.compile(r'원스톱\s*[:：]\s*(?P<hospital>[^\n]+)')
```

### TBSS 매핑
- **HearingLossDetail.examTargetHospital** (특진병원) ← `원스톱:`
- **HearingLossExam.examScheduledAt** ← (일정방 메시지로 보강)
- **Case.kwcOfficeName** ← `원스톱:`이 "근로복지공단 ○○병원/○○의원"이면 그 지역에서 추정
- **신규 양식 자동완성 후보**: 진찰요구서 PDF 100건을 OCR 골드셋으로 사용해 `lib/ocr/parsers/` 회귀 테스트 가능

## B. 일정방 — 특진/심사 일정 동기화 (정형도 96%)

### 패턴
```
<{병원명} {상병} 특진 일정[ 변경| 취소]>

{재해자명}
{TF}  ({담당자})
1차 : 2026.05.06.(수) 14:00 {참석자}
2차 : 2026.05.07.(목) 14:00
* 변경 사유 : {사유}
```

- 헤더 `<...일정/변경/취소>` 매치: **192/200 (96%)**
- 병원 top: 군산의료원 18, 근로복지공단 대구병원 17, 서울적십자병원 15, 성가롤로병원 9, 현대유비스병원 7, 근로복지공단 부산의원 7
- TF 등장 분포: 여수 9, 순천 8, 창원 8, 대구수성 6, 대구달서 6 (한 노무사 시야 기준)

### 정규식 후보
```python
PAT_일정헤더 = re.compile(r'<(?P<hospital>[^ >]+)\s*(?P<disease>난청|폐질환|진폐|근골[가-힣]*)\s*특진\s*일정(?P<variant>\s*변경|\s*취소)?>')
PAT_회차 = re.compile(
    r'(?P<round>\d+)차\s*[:：]\s*'
    r'(?P<date>\d{4}[.년]\s*\d{1,2}[.월]\s*\d{1,2})[.\s]*'
    r'\((?P<dow>[월화수목금토일])\)\s*'
    r'(?P<time>\d{1,2}:\d{2})?',
)
```

회차 패턴은 표기 흔들림(예: `1차 일정 종료 : 25. 10. 23. (목)`, `* 1차 일정 : 05.06. (수)`) 때문에 정규식만으로 24건 매치. **1차 패스 정규식 + 2차 LLM 보정** 권장.

### TBSS 매핑
- **HearingLossExam** 회차별 `examScheduledAt`, `examHospital`, `examRound`, `examSet` (INITIAL/RE/RE2)
- 변경/취소 메시지는 *상태 자동 갱신 트리거*로 사용:
  - `<일정 변경>` → 기존 row 업데이트
  - `<일정 취소>` → status="CANCELLED"
- 신규 페이지 권고: `/calendar/exams` (특진 일정 통합 캘린더) — 일정방 메시지를 단일 진실 소스로

## C. 폐질환 통합방 — 직업력 + 처분 학습 GOLD

### 패턴
```
{재해자명} 님 {상병=copd|진폐 등} {불승인|승인} 자료 전달드립니다.

- 적용사업장 : {사업장명}
- 직종 : {직종 (콤마/슬래시 다중)}
- 직력 : {기간 (X년 Y개월 Z일)}
```

- "`적용사업장 :`" 정확 매치 45 / 텍스트 111
- "`직종 :`" 45 (사업장과 동기화)
- 처분: **불승인 32 / 승인 18** (불승인이 1.7배) → 직업력 정밀도가 결과 가르는 영역

### 정규식 후보
```python
PAT_사업장 = re.compile(r'적용사업장\s*[:：]\s*(?P<site>[^\n]+)')
PAT_직종   = re.compile(r'직종(?:/직력)?\s*[:：]\s*(?P<job>[^\n]+)')
PAT_직력   = re.compile(r'직력\s*[:：]\s*(?P<duration>[^\n]+)')
PAT_상병   = re.compile(r'상병명?\s*[:：]\s*(?P<disease>[^\n]+)')
PAT_처분   = re.compile(r'(불승인|승인|기각|반려|일부승인)\s*자료')
```

### TBSS 매핑
- **Case.workHistory** (JSON 배열): 사업장+직종+직력 → 한 행으로 자동 인입 가능
- **Case.workHistoryRaw** : 원본 메시지 텍스트 보존 (소스 추적)
- **Case.status / closedReason** : 처분 결과로 추정 가능 (REJECTED / APPROVED)
- **신규 페이지 권고**: 직업력 자동 초안 입력 (메시지 붙여넣으면 파싱)

## D. 중증&유족 공유방 — 공단 Web발신 SMS 파서

### 패턴 (공단 발송 SMS, 채팅방에 forward됨)
```
[Web발신]
{이름}*고객님(사건번호 2026-XXXXXXX)께서 요양급여 신청하신 질병은
{날짜}({요일}) {시각} {지역} 업무상질병판정위원회에서 심의할 예정입니다.
지사에서 ...
```

- Web발신 SMS: **31/153**
- 사건번호 추출 가능: **31/31 (100%)**
- 질판위 지역 분포: 서울남부 22 (압도적), 경인북부 2, 대구 2, 서울북부 2, 부산 2, 경남 1

### 추가 패턴 (자체 작성 메시지)
```
<{지역} 질판위 심의 일정>

{이름} ({TF / 담당자}) : 26.05.12.(화) 15시
{상병}
상담소 {담당자명}
```

### 정규식 후보
```python
PAT_사건번호 = re.compile(r'사건번호\s*(?P<no>\d{4}-\d{6,7})')
PAT_질판위지역 = re.compile(r'(?P<region>[가-힣]+)\s*업무상질병판정위원회')
PAT_심의일시 = re.compile(
    r'(?P<date>\d{4}[.\-]\d{2}[.\-]\d{2})\s*'
    r'\(?(?P<dow>[월화수목금토일])(?:요일|요)?\)?\s*'
    r'(?P<time>\d{1,2}[:시]\d{0,2})?',
)
```

### TBSS 매핑
- **Case.caseNo** ← 사건번호
- **Case.qpwSchedule** (신규 필드 후보) ← 질판위 심의 일시·지역
- **자동 알림**: 질판위 일정 D-3 / D-1 알림 자동 발송 (현재 `/api/telegram/cron-notify` 인프라 활용)
- **공단 SMS 자동 인입**: 노무사가 메시지 forward만 해도 사건과 자동 매칭 가능 (사건번호 키)

## E. 근골 접수방 — ICD 코드 + 해시태그 + 직업력

### 패턴
```
#{근골요양승인자료|근골요양불승인자료|근골장해승인자료|...}
{재해자명}
{TF}) {담당자}님 건
{재해자명} 님 근골격계({부위}) {요양|장해}{승인|불승인} 자료 전달드립니다.

- 적용사업장 : {사업장}
- 직종/직력 : {직종} 약 {기간}
- 상병명 : {상병명}, {부위}[{ICD}]
```

- 해시태그: `근골요양승인자료` 16, `근골요양불승인자료` 13, `근골장해승인자료` 7
- ICD 분포: **M751 (회전근개) 15, M511 (요추간판탈출) 12, M171 (무릎 골관절증) 7, M4806 (요추 척추증) 6, M170 (양측 슬관절 OA) 6**
- 직종/직력 정확 매치 68건 / 텍스트 73건 = **93%**

### 정규식 후보
```python
PAT_해시태그 = re.compile(r'#(?P<tag>근골[가-힣]+(?:승인|불승인|장해|요양)[가-힣]*)')
PAT_ICD     = re.compile(r'\[(?P<icd>[A-Z]\d{2,4})\]')
PAT_직종직력 = re.compile(r'직종(?:/직력)?\s*[:：]\s*(?P<job>[^\n]+?)\s*약\s*(?P<duration>\d+년(?:\s*\d+개월)?(?:\s*\d+일)?)')
```

### TBSS 매핑
- **Case.diseaseCode** (신규 필드 후보) ← ICD 코드
- **Case.workHistory** : 직종/직력 단일 행 자동 인입
- **장해등급 페이지** (`app/grade/`) : ICD-기반 부위 자동 매핑 (예: M751 → 어깨회전근개)

## 종합 인사이트

### 1. 메시지 정형도 매우 높음 → 정규식 1차 + LLM 보정 충분
- 진찰요구서: 99%
- 일정방 헤더: 96%
- 근골 직종/직력: 93%
- 폐질환 사업장+직종: 동기화율 100%
- 공단 SMS 사건번호: 100%

→ Document AI / Tesseract OCR 없이도 텍스트 메시지만으로 *지금 즉시* 자동 인입 가능한 영역이 크다.

### 2. 가장 큰 자동화 ROI 우선순위
1. **특진 일정 통합 캘린더** (일정방) — 정형도 96%, 즉시 구현 가능, 노무사 다수 사용
2. **공단 SMS 사건번호 자동 매칭** (중증&유족방) — 100% 추출, 노무사 forward만으로 사건 동기화
3. **직업력 자동 초안** (폐질환·근골 접수방) — 사업장/직종/직력 정형 입력
4. **진찰요구서 OCR 회귀 테스트셋** (진찰요구서 방 PDF 100건) — 신뢰도 측정 골드셋

### 3. 채팅방 ↔ 상병 매핑
- 진찰요구서 방 = 사실상 *난청 전용* (95%)
- 일정방 = *난청 특진 위주* (병원이 모두 난청 특진 거점)
- 폐질환 통합방 = COPD 위주
- 근골 접수 = 회전근개·요추 위주

→ 신규 페이지 설계 시 *상병별 분기*가 채팅방 흐름과 일치 (현재 `HearingLossDetail` / 폐질환 / 근골 분리 설계와 부합).
