**432건 모두 동일 시트 구조**:

| 시트 | 의미 |
|------|------|
| 0. 이용방법 | 양식 사용 안내 |

**핵심 입력 필드 (사건당 100% 입력)**:

- 의료기관 리스트: 가천대 길병원, 가톨릭대 대전성모, 가톨릭대 성빈센트, 가톨릭대 의정부성모/인천성모, 강동경희대, 강동성심, 강릉아산, 강북삼성, 강원대, 거붕백, 건국대 충주, 검단탑, 경북대, 경희의료원, 계명대 동산, 고려대 안산/구로, 고신대 복음, 광주기독, 구미강동/차, 국립경찰, 군산의료원, 근로복지공단 (대구/대전/동해/광주의원) 등 50+ 의료기관 사전 등록됨
- TF 사전 등록: 창원TF, 의정부TF, 문경TF, 울산TF, 부산TF, 대구TF, 포항TF, 양산TF
- 담당자 사전 등록: (현재 시스템 사용자 13명+) — 자료입력서식에서 노출됨

→ **TBSS Case import 도구를 만들면**: 폴더당 1개 자료입력서식 → 1개 Case 자동 생성 가능. 432건 즉시 import 가능.

## 5. 권고: TBSS 사건 import 도구 설계

### 5.1 아키텍처
```
[자료입력서식 .xls 업로드]
   ↓
[openpyxl 시트 1.입력 / 3.대리인 / 4.특별&전문 파싱]
   ↓
[필드 매핑]
   - 근로자 → Patient (rrn 중복 방지: upsert)
   - 사건종류 → Case.caseType
   - 대리인 → Case.staff (직무개시번호 매칭)
   - 특별진찰 → HearingLossExam (caseType이 난청일 때만)
   ↓
[검증]
   - 필수 필드 누락 시 사용자에게 prompt
   - 중복 사건 (Patient + caseType) 검사
   ↓
[INSERT Case + Patient (필요 시) + 관련 detail 생성]
```

### 5.2 우선 적용 대상
- **0. 최초팀 공유 폴더의 자료입력서식 432건** → 일괄 import
- 마스터 DB (포항/대구/창원 등 7개 지사) → 사건 진행상황 동기화

### 5.3 신규 Prisma 필드 권고
```prisma
model Case {
  // 기존 필드 +
  introducer        String?  // 소개자 (자료입력서식)
  salesStaff        String?  // 영업담당자
  caseChannel       String?  // 사건경로 (어떻게 들어온 사건인지)
  preliminaryConsult String? // 예비 문진 결과
}
```
