## 1. 브랜드 컬러 팔레트

로고에서 추출한 3가지 원색을 기반으로 시스템 전체 컬러를 구성한다.

### Primary — 스카이블루 (로고 원색 `#29ABE2`)
```
--blue-700: #1480B0   /* 진한 강조 */
--blue-600: #1A95C8   /* 호버 */
--blue-500: #29ABE2   /* 로고 원색 — Primary 버튼, 포커스 링 */
--blue-400: #50BDEA   /* 사이드바 활성 라인, 아이콘 */
--blue-300: #84D1F2
--blue-100: #DCEEFA   /* 배경 틴트, 뱃지 배경 */
--blue-50:  #EEF7FD   /* 테이블 hover, 연한 배경 */
```

### Secondary — 딥그린 (로고 원색 `#006838`)
```
--dkgreen-800: #004025
--dkgreen-700: #005530   /* 헤더 타이틀, 재해자명 */
--dkgreen-600: #006838   /* 로고 원색 — 사이드바, 페이지네이션 active */
--dkgreen-500: #00854A   /* hover */
--dkgreen-100: #D0EAD9   /* 뱃지 배경 */
--dkgreen-50:  #E8F5EE
```

### Accent — 라임그린 (로고 원색 `#8DC63F`)
```
--green-700: #5A8A1F
--green-600: #6EA02A   /* hover */
--green-500: #8DC63F   /* 로고 원색 — 신규등록·긍정 액션 버튼, 승인완료 */
--green-400: #A2D158
--green-100: #E8F5D0   /* 뱃지 배경 */
--green-50:  #F4FAE8
```

### Semantic (상태 컬러)
```

위험/불승인:   #C0392B (bg: #FDEDEC)
주의/서류준비: #B45309 (bg: #FEF3C7)
```

### Neutrals

## 2. 레이아웃 구조

```
--sidebar-width:  220px
--header-height:  56px
--content-padding: 22px 24px
--radius-sm: 4px    /* 버튼, input */
--radius-md: 6px    /* 카드, 패널, 테이블 */
--radius-lg: 10px   /* 모달 */
--shadow-sm: 0 1px 3px rgba(0,0,0,0.07)
--shadow-md: 0 2px 8px rgba(0,0,0,0.09)
```

**전체 레이아웃:** 사이드바(고정) + 메인(flex-grow)  
**메인 구성:** 헤더(고정) + 콘텐츠(스크롤)

- 우측 퀵 링크 패널: 북마크 폴더 형태 (클릭 시 펼쳐지는 방식, 세로 늘어뜨림 금지)
- 현재 웹은 데스크톱 우선 (모바일 최적화 미적용)

## 6. 페이지별 UI 스펙

### 6-1. 로그인 페이지
- 상단: 대표 인사말 박스(좌) + 관리자 공지사항 박스(우)
- 하단: 로그인 입력창 (가로로 넓게, 중앙 정렬)
- [가입 요청] 버튼: 로그인 버튼 하단

### 6-2. To Do List
- 좌측 1/2: 달력 그리드 + 날짜별 To Do 이력
- 우측 1/2 상단: 요청 유형별 필터 버튼
- 우측 1/2 하단: 기간 설정 조회
- 하단 여백: 자유 기술 메모 공간

### 6-3. 사건 목록 (`/cases`)
- 상단: 다중 필터 (상병, 진행단계, 담당자, TF, 지사)
- 테이블: 이름, 주민번호, 상병, 진행상황(한글), 담당자, TF, 수임일
- 진행상황: 한글 로컬라이징 적용 (영문 Enum 표시 금지)
- 행 클릭: 사건 상세로 이동

### 6-4. 사건 상세
- 상단: 재해자 기본 정보 (이름, 주민번호, 연락처, 주소)
- 탭 구성: 사건 정보 / 직업력 / 특진 / 전문조사 / 결정 / 이의제기
- 타임라인: 우측 or 하단 (진행 이력 표시)
- 하단: 보험급여 서식 생성 버튼

### 6-5. 직업력 테이블
- 근무기간 컬럼 표시
- 일용직 20일 미달 시 갈색 강조 (`isDailyWorker` 연동)

### 6-6. 특진 (HearingLossExam)
- 기본: 1~3차 + +N차 버튼으로 동적 추가
- 재특진: 기본 숨김 → 필요 시 펼치기
- 각 회차: 검사일, 담당자, 청력 수치 (기도/골도/어음), ABR, 임피던스, 신뢰성

### 6-7. 지사장 관리·운영 (조직관리자 전용)
- 대시보드 형태 (카드 + 차트)
- 매출/비용: 막대 차트 (월별/분기별/연도별 토글)
- 외근직 실적: 테이블 + 개인별 수치
- 내근직 업무 부하: 게이지 또는 히트맵
- 업무 지시 게시판: 기한 표시, 긴급/일반 구분

### 6-8. RAG 검색 페이지 (`/rag-search`, Phase 6)
```
[검색창]
"조적공 근골격계 승인 사례 찾아줘"  [검색]

[검색 범위 필터] 전체 / 승불자료 / 지사운영자료 / 지침·논문
[상병 필터] 소음성난청 / 근골격계 / 진폐 / 전체

[결과 영역]
  📋 AI 요약 (Claude API 생성)
  📄 관련 파일 (구글 드라이브 링크 + 열기 버튼)
```

## 7. 디자인 원칙

1. **브랜드 통일성** — 홍보물·로고와 동일한 3색 체계 유지
2. **눈 피로 최소화** — 쨍한 원색 직접 배경 사용 금지. 배경은 gray-50/white, 원색은 포인트·버튼·라인에만
3. **전문성** — 과도한 애니메이션·장식 배제. 정보 밀도와 가독성 우선
4. **상태 명확성** — 사건 상태는 반드시 배지 컬러로 즉시 식별 가능하게
5. **일관성** — 동일 의미의 컴포넌트는 전 페이지 동일 스타일 적용
6. **한글 우선** — 모든 상태값, 라벨은 한글로 표시 (Enum 영문 노출 금지)
7. **기존 스타일 유지** — 인라인 스타일 / Tailwind 혼재 현황 그대로 존중
