## 1. 권한 체계

### 권한 Enum

| 권한 | 대상 | 설명 |
|------|------|------|
| `ADMIN` | 시스템 관리자 (이정준) | 전체 접근 + 관리자 페이지 + 계정 관리 |
| `조직관리자` | 대표, 이사, 권역지사장, 지사장, 센터장, 연구원장, 법률원장, 수석노무사 | 지사장 관리·운영 페이지 포함 |
| `STAFF` | 노무사, 내근직, 외근직 | 일반 업무 기능 전체 |
| `이산계정` | 노무법인 이산 임직원 | 사건 조회 페이지만 접근 |

### 계정 생성 방식
- 자발적 가입 불가
- 사용자가 가입 요청 → ADMIN이 계정 생성 후 전달
- 계정 형식: `이니셜3자리+입사순번@thebosang.kr`

## 3. 데이터 접근 제한

### 이산계정 제한
- 사건 조회 페이지: 요약 정보만 표시 (상병, 성명, 주민번호, 연락처, 진행경과, 담당 지사, TF, 노무사)
- 사건 상세 페이지: **접근 불가** (라우팅 차단)
- 본인 영업 사건만 조회 가능 (TF 필터 자동 적용)

### STAFF 제한
- 지사장 관리·운영 페이지: 접근 불가
- 다른 지사 민감 데이터 조회 제한 (추후 구현)

### 주민번호 처리
- DB 저장: 전체 주민번호 저장
- 화면 표시: 기본 마스킹 (`000000-*******`)
- 마스킹 해제: ADMIN / 조직관리자 / STAFF만 가능 (이산계정 불가)
- API 응답: 이산계정 요청 시 주민번호 마스킹 처리 필수

## 4. 인증 흐름 (Auth.js v5)

```
사용자 로그인 요청
  → POST /api/auth/signin
  → Auth.js v5 (NextAuth) 세션 생성
  → 세션 쿠키 발급

페이지 접근
  → middleware.ts 에서 세션 확인
  → 권한 체크 (role 필드)
  → 권한 없으면 /login 또는 /unauthorized 리다이렉트

API 접근
  → auth() 호출로 세션 확인
  → session.user.role 기반 분기
```

### 중요 설정
- `AUTH_URL` 환경변수: Railway 프로덕션 URL로 반드시 설정
  - 미설정 시 로그아웃이 localhost로 리다이렉트되는 버그 발생
- 세션 만료: 기본값 사용 (추후 조정 필요 시 협의)

## 5. API 보안 규칙

### 모든 API 라우트 필수 처리
```typescript
// 모든 API Route에서 세션 확인 필수
const session = await auth()
if (!session) return Response.json({ error: 'Unauthorized' }, { status: 401 })

// 권한별 분기 예시
if (session.user.role === '이산계정') {
  // 이산계정 제한 처리
}
```

### 이산계정 API 응답 처리
```typescript
// 주민번호 마스킹 필수
const maskedPatient = {
  ...patient,
  ssn: session.user.role === '이산계정'
    ? patient.ssn.replace(/(\d{6})-?(\d{7})/, '$1-*******')
    : patient.ssn
}
```

## 6. 텔레그램 Webhook 보안

### 수신 검증
```typescript
// Telegram-Bot-Token 헤더 또는 secret token 검증
const telegramToken = req.headers.get('X-Telegram-Bot-Api-Secret-Token')
if (telegramToken !== process.env.TELEGRAM_WEBHOOK_SECRET) {
  return Response.json({ error: 'Forbidden' }, { status: 403 })
}
```

### 환경변수
- `TELEGRAM_BOT_TOKEN`: 봇 API 토큰 (Railway 환경변수)
- `TELEGRAM_WEBHOOK_SECRET`: webhook 검증용 시크릿
- 절대 클라이언트 코드에 노출 금지

## 7. NAS 연결 보안 (Phase 6)

### 접근 방식
- WebDAV HTTPS 연결 권장
- 인덱싱 전용 읽기 전용 계정 사용 (쓰기 권한 불필요)
- 환경변수 관리:
  - `NAS_WEBDAV_URL`: NAS WebDAV 엔드포인트
  - `NAS_USERNAME`: 읽기 전용 계정
  - `NAS_PASSWORD`: Railway 환경변수로만 관리 (코드 하드코딩 금지)

### 네트워크 접근
- Railway 서버 → NAS 직접 접근: 포트포워딩 또는 VPN 필요
- NAS 포트: 기본 WebDAV는 5005 (HTTP) / 5006 (HTTPS)

## 8. 개인정보 처리 원칙

### 민감 정보 목록
- 주민등록번호 (전체)
- 연락처
- 주소
- 진단명 / 장해등급 / 보상금액

### 처리 원칙
1. 최소 수집: 업무에 필요한 정보만 수집
2. 접근 제한: 권한에 따른 데이터 노출 수준 차별화
3. 이산계정: 상세 의료 정보 / 보상금액 노출 불가
4. 로그: 민감 데이터 조회 로그 추후 구현 검토
