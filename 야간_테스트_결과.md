## 4. 신뢰성 미달 잔존 이슈

### 4.1 김옥자 고용보험 일용직 48 vs 49 (1/5 변동)
- 일용직 행이 많은 경우 ±1건 over-extract 가능성
- 더 강화: 출력 후 검증 단계, "자격이력 표 행 절대 일용직에 포함하지 마라" 강조 (이미 적용됨)
- 잔존 시 **Sonnet 폴백** 또는 multiple Haiku + voting 고려

### 4.2 안세윤 헤비 케이스 rate limit
- Anthropic Tier 1: 50K input TPM, 10K output TPM (Haiku)
- 청크 6개 × ~7K input + 일용직 643건 × ~50 output = 한도 초과
- 해결책 후보:
  1. **Anthropic Tier 2 업그레이드** ($40 누적) — Tier 1의 1.6배
  2. Sequential 처리 + retry 8/16/32/64s (이미 코드에 적용)
  3. Document AI Batch async (GCS 버킷 추가 설정 필요)

## 5. 헤비 케이스 권장 정책

10초 절대 불가능. 권장 UX:

**현재 적용된 것**: 청크 수에 따른 동시성 자동 조정
- ≤3 청크: 동시성 3 (작은 PDF)
- 4-6 청크: 동시성 2 (중간 PDF)
- 7+ 청크: 동시성 1 (헤비, sequential)

**아침 회의 후 결정 필요**:

**A안 — Tier 2 업그레이드**: $40 결제로 헤비 케이스 안정화. 가장 단순.

**B안 — UX 개선**: 페이지 수에 따라 추정 시간 표시 ("약 3분 소요")

**C안 — Document AI Form Parser**: 표 구조 보존, 김옥자 국민연금 같은 케이스 정확도 ↑. GCS bucket 설정 필요.
