## 세션 타입: 기능 개발 세션
## 작업 개요
관리자 페이지(/admin)에 '지사 관리' 탭을 추가한다.
Branch 모델을 신규 생성하여 지사 기본정보, 담당 TF 배정, 소속 인원, 권역 배정을 DB로 관리한다.
기존 하드코딩 상수(TF_BY_BRANCH, REGION_BRANCHES 등)를 Branch DB 기반으로 전환한다.
BranchSelector를 Branch DB에서 로드하도록 변경한다.

- 기존 BranchSelector/ContactSelector 인터페이스 변경 최소화

## Step 1: Branch 모델 생성

### 1-1. prisma/schema.prisma에 Branch 모델 추가

```prisma
model Branch {
  id            String   @id @default(cuid())
  
  // 기본정보
  name          String   @unique    // 지사명 (예: "울산지사", "부산경남지사")
  shortName     String?             // 약칭 (예: "울산", "부산경남")
  address       String?             // 주소
  phone         String?             // 전화번호
  fax           String?             // 팩스번호
  
  // 권역 배정
  region        String?             // 권역명 (수도권역/부울경남권역/대구경북권역/전라권역)
  
  // 담당 TF 배정 (JSON 배열)
  assignedTFs   Json?               // ["더보상울산TF", "이산울산북부TF", "이산울산남부TF"]
  
  // 소속 인원 (Contact.branch와 매칭)
  branchManagerId  String?          // 지사장 Contact.id (문자열 참조, FK 아님)
  
  // 법인 구분
  firmType      String   @default("TBOSANG")  // TBOSANG / ISAN
  
  // 상태
  isActive      Boolean  @default(true)
  displayOrder  Int      @default(0)
  
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@map("branches")
}
```

## Step 2: Branch CRUD API

### 2-1. `app/api/branches/route.ts` (GET + POST)

```typescript
import { NextResponse } from 'next/server'
import { prisma } from '@/lib/prisma'

// GET: 지사 목록 조회
export async function GET(request: Request) {
  const { searchParams } = new URL(request.url)
  const firmType = searchParams.get('firmType')
  const region = searchParams.get('region')
  const includeStaff = searchParams.get('includeStaff') === 'true'

  const where: any = { isActive: true }
  if (firmType) where.firmType = firmType
  if (region) where.region = region

  const branches = await prisma.branch.findMany({
    where,
    orderBy: { displayOrder: 'asc' },
  })

  // 소속 인원 수 조회 (Contact.branch 기준)
  if (includeStaff) {
    const enriched = await Promise.all(
      branches.map(async (b) => {
        const staffCount = await prisma.contact.count({
          where: { branch: b.name },
        })
        return { ...b, staffCount }
      })
    )
    return NextResponse.json({ branches: enriched })
  }

  return NextResponse.json({ branches })
}

// POST: 지사 등록
export async function POST(request: Request) {
  const body = await request.json()
  const branch = await prisma.branch.create({ data: body })
  return NextResponse.json(branch)
}
```

### 2-2. `app/api/branches/[id]/route.ts` (GET + PATCH + DELETE)

```typescript
import { NextResponse } from 'next/server'
import { prisma } from '@/lib/prisma'

// GET: 지사 상세 (소속 직원 목록 포함)
export async function GET(
  request: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params
  const branch = await prisma.branch.findUnique({ where: { id } })
  if (!branch) return NextResponse.json({ error: 'Not found' }, { status: 404 })

  // 소속 직원 목록 (Contact.branch 매칭)
  const staff = await prisma.contact.findMany({
    where: { branch: branch.name },
    orderBy: [{ jobGrade: 'asc' }, { displayOrder: 'asc' }],
    select: { id: true, name: true, title: true, jobGrade: true, mobile: true, email: true, hireDate: true, leaveDate: true, userId: true },
  })

  return NextResponse.json({ branch, staff })
}

// PATCH: 지사 수정
export async function PATCH(
  request: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params
  const body = await request.json()
  const branch = await prisma.branch.update({ where: { id }, data: body })
  return NextResponse.json(branch)
}

// DELETE: 지사 비활성화 (soft delete)
export async function DELETE(
  request: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params
  await prisma.branch.update({ where: { id }, data: { isActive: false } })
  return NextResponse.json({ ok: true })
}
```

### 5-1. TF → 지사 역방향 조회 API 추가

`app/api/branches/by-tf/route.ts` 생성:

```typescript
import { NextResponse } from 'next/server'
import { prisma } from '@/lib/prisma'

// GET /api/branches/by-tf?tfName=더보상울산TF
export async function GET(request: Request) {
  const { searchParams } = new URL(request.url)
  const tfName = searchParams.get('tfName')
  
  if (!tfName) {
    return NextResponse.json({ error: 'tfName required' }, { status: 400 })
  }
  
  // 모든 branch의 assignedTFs에서 해당 TF를 포함하는 지사 찾기
  const branches = await prisma.branch.findMany({
    where: { isActive: true },
  })
  
  const matched = branches.find(b => {
    const tfs = (b.assignedTFs as string[]) || []
    return tfs.includes(tfName)
  })
  
  return NextResponse.json({ branch: matched || null })
}
```

### 5-2. 전체 TF 목록 API

`app/api/branches/all-tfs/route.ts` 생성:

```typescript
import { NextResponse } from 'next/server'
import { prisma } from '@/lib/prisma'

// GET /api/branches/all-tfs → 지사별 TF 매핑 반환
export async function GET() {
  const branches = await prisma.branch.findMany({
    where: { isActive: true },
    orderBy: { displayOrder: 'asc' },
    select: { name: true, shortName: true, region: true, assignedTFs: true },
  })
  
  // TF_BY_BRANCH 형태로 변환
  const tfByBranch: Record<string, string[]> = {}
  const regionBranches: Record<string, string[]> = {}
  
  branches.forEach(b => {
    tfByBranch[b.name] = (b.assignedTFs as string[]) || []
    if (b.region) {
      if (!regionBranches[b.region]) regionBranches[b.region] = []
      regionBranches[b.region].push(b.name)
    }
  })
  
  return NextResponse.json({ tfByBranch, regionBranches, branches })
}
```

```typescript
    const { prisma } = await import('@/lib/prisma')
    const branches = await prisma.branch.findMany({
      where: { isActive: true },
      orderBy: { displayOrder: 'asc' },
    })
    const map: Record<string, string[]> = {}
    branches.forEach(b => {
      map[b.name] = (b.assignedTFs as string[]) || []
    })
    return Object.keys(map).length > 0 ? map : TF_BY_BRANCH // fallback
  } catch {
    return TF_BY_BRANCH // fallback to hardcoded
  }
}
```

**주의**: 기존 TF_BY_BRANCH 상수는 삭제하지 않는다. DB 로드 실패 시 fallback으로 사용.

## 주의사항

1. **이중 DB 주의**: Branch 모델은 메인 DB (schema.prisma)에 생성. User(auth.prisma)와 무관.
2. **assignedTFs는 Json 타입**: Prisma에서 Json으로 저장, 조회 시 `as string[]` 캐스팅 필요.
3. **기존 상수 fallback 유지**: DB 로드 실패 시 기존 하드코딩 상수로 동작해야 함. 상수 파일 삭제 금지.
4. **Contact.branch 문자열 매칭**: Branch.name과 Contact.branch가 정확히 일치해야 소속 인원 조회 가능. 불일치 시 Contact.branch 업데이트 필요.
5. **displayOrder**: 시드 데이터의 순서 = 화면 표시 순서. 관리자가 수정 가능.
6. **Next.js 15 동적 라우트**: `params: Promise<{ id: string }>` + `await params` 필수.
