# Railway 포트 바인딩 수정

## 원인
Railway는 컨테이너 시작 시 PORT 환경변수(예: 8080)를 주입하는데
`next start`가 해당 포트를 읽지 못하고 기본 3000번으로 실행됨
→ Railway 헬스체크가 지정 포트로 요청해도 응답 없음 → 배포 실패

## 수정: package.json start 스크립트

파일: package.json

아래 "start" 항목을 찾아서 수정:

```json
// 수정 전
"start": "next start"

// 수정 후
"start": "next start -p ${PORT:-3000}"
```

### 실제 수정 명령

```bash
# package.json의 start 스크립트 확인
grep -n '"start"' package.json

# str_replace로 수정
```

package.json에서:
```
"start": "next start"
```
→
```
"start": "next start -p ${PORT:-3000}"
```

## 수정 후 커밋

```bash
git add package.json
git commit -m "fix: Railway PORT 환경변수 바인딩 (next start -p \${PORT:-3000})"
git push origin main
```

Railway가 자동 재배포 → 헬스체크 통과 확인.
