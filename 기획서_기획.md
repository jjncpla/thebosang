**NAS 연결 아키텍처**
```
Synology NAS (2TB, 법인 자료)
  ↓ WebDAV API 또는 SMB
Railway 서버 (Python 인덱서 스크립트)
  ↓ 청크 분할 + 임베딩 (OpenAI text-embedding-3-small)
pgvector (Railway PostgreSQL)
  ↓
TBSS /rag-search 페이지 + 텔레그램 봇 연동
```

## 2. 페이지별 기획 상세

### 2-1. 로그인 페이지
- 상단: 대표 인사말(좌) + 관리자 공지사항(우) 박스
- 하단: 로그인 입력창 (가로로 넓게)
- [가입 요청] 버튼 → 관리자 페이지로 요청 전달
- 로그인 성공 → To Do List 이동

### 2-2. To Do List (로그인 후 첫 화면)
- 좌측 1/2: 달력 + 날짜별 To Do 이력
- 우측 상단: 요청 유형별 필터 (정산 요청, 이의제기 작성 요청, 제척 임박 등)
- 우측 하단: 기간 설정 조회
- 하단 여백: 자유 기술 메모
- 이의제기 기일 관리 → 제척 도래 시 자동 반영

### 2-3. 사건 관리
- 병원 환자 관리 프로그램 컨셉
- 재해자(Patient) 고정정보 기준 조회·관리
- 기획사건: 소음성 난청 / COPD / 진폐 (초진→특진→전문조사→장해급여)
- 일반산재: 근골격계, 업무상 사고, 직업성 암, 유족 등
- 스키마 상세 → `docs/기획서_개발.md` 섹션 3 참조

### 2-4. 사건 조회 (이산·외부)
- 대상: 이산 임직원 + 더보상 영업담당자 (본인 영업 사건만)
- 표시: 상병, 성명, 주민번호, 연락처, 진행경과, 담당 지사, TF, 노무사
- 목록 PDF 저장 기능

### 2-5. 이의제기 관리
- 기일 관리: 제척기간 산정·모니터링 → To Do List 자동 알림
- 이유서·의견서: 사건 선택 시 재해자 정보 자동 반영 → PDF 생성
- 우측 패널: 관련 사례·논문 (RAG-5 단계에서 구현)

### 2-6. TF 업무
- 담당TF 모니터링: 텔레그램방 webhook 연동
- TF 공지: 공지 서식 자동 작성 → 공지방 발송
- TF 통계: webhook 데이터 + 사건 DB 연계 시각화

### 2-7. 지사장 관리·운영 (조직관리자 전용)
- 접근: 조직관리자 이상
- 구성: 매출·비용, 외근직 실적, 내근직 업무 부하, 홍보 현황, 운영규정 조회, 업무 지시 게시판
- 인사 평가·승진 대상자 여부 판단 기능 포함
- 각 항목별 인입 DB 상세 → 상세 개발 시 별도 전달

### 2-8. 텔레그램 양방향 연동 상세 (Phase 4)

**텔레그램 → TBSS 방향**
| 기능 | 동작 방식 |
|------|----------|
| 키워드/패턴 감지 → DB 자동 저장 | webhook 수신 → 정규식 파싱 → DB INSERT |
| 메시지 → 사건 데이터 입력 트리거 | 특정 커맨드 감지 → TBSS 입력 폼 자동 열기 또는 API 호출 |

**TBSS → 텔레그램 방향**
| 기능 | 동작 방식 |
|------|----------|
| 기일/To Do 알림 자동 발송 | TBSS 크론잡 → 텔레그램 Bot API sendMessage |
| 사건 상태 조회 응답 | 봇 질의 → Claude API 자연어 처리 → DB 조회 → 응답 |

## 3. 현업 업무 방식 (현재 문제점 → 개선 방향)

| 현재 방식 | 문제점 | 개선 방향 |
|-----------|--------|-----------|
| NAS 엑셀로 사건 관리 | 파일 간 연결 없음, 동시 접속 서버 에러, 수작업 과다 | TBSS DB로 통합 |
| 텔레그램으로 문서 공유 | 파일 저장 불완전, 필터링 어려움 | webhook 양방향 연동 (Phase 4) |
| 한글 메일머지로 접수서식 생성 | 수작업, 파편화 | TBSS 자동 서식 생성으로 대체 |
| 통계 수작업 | 광역 통계 불가 | DB 기반 자동 집계·시각화 |
