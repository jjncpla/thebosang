## 2. 개발 원칙 (반드시 준수)

1. **수술적 수정**: 기존 UI·로직 건드리지 않고 필요한 부분만 변경
2. **전체 재작성 금지**: 명시적 요청 없으면 기존 파일 전체 재작성 안 함
3. **기존 스타일 유지**: 인라인 스타일 / Tailwind 혼재 현황 그대로 존중
4. **TS 상수로 데이터 관리**: 정적 데이터는 동적 로딩 대신 TS 상수 하드코딩
5. **긴 지시사항은 .txt 파일로**: Claude Code CLI 터미널 붙여넣기 한계 대응
6. **임의 패키지 설치 금지**: 사전 협의 없이 새 패키지 추가 금지

## 3. 사건관리 PRD (핵심 스키마)

### 3-1. 핵심 설계 원칙

- **Patient(재해자)** 가 최상위 엔티티. 1명이 복수 상병(사건) 가능
- **Case(사건)** 은 항상 특정 상병에 귀속. 상병이 다르면 별개 사건
- **파생 사건**: 평균임금 정정 청구 시 원본 `caseId`를 `parentCaseId`로 연결
- **B방식 FK**: `Case → ObjectionReview → ObjectionCase` 명시적 `caseId` FK

### 3-2. 상병 계열

- **기획사건**: 소음성 난청 / COPD / 진폐증
- **일반산재**: 근골격계 / 업무상 사고 / 직업성 암 / 유족 / 기타

### 3-3. CaseStatus Enum (기획사건용)

```
CONSULTING              상담 중 (초진 전, 약정 전)
CONTRACTED              약정 완료 / 진행 개시
DOC_COLLECTING          기본자료 수집 중
SUBMITTED               공단 접수 완료 (우편 발송)
EXAM_REQUESTED          특진진찰요구서 수령
EXAM_CLINIC_SELECTED    특진병원 선택확인서 제출 완료
EXAM_SCHEDULED          특진 일정 확정
IN_EXAM                 특진 진행 중
EXAM_DONE               특진 완료 (3차 종료 + 의무기록지 수령)
EXPERT_REQUESTED        전문조사진찰요구서 수령
EXPERT_CLINIC_SELECTED  전문조사기관 선택 완료
EXPERT_DONE             전문조사 완료
BANK_REQUESTED          통장사본 요청 수령 (전화)
BANK_SUBMITTED          통장사본 제출 완료
DECISION_RECEIVED       장해등급결정통지서 수령
REVIEWING               노무사 검토 중 (평정/적사변 실익)
INFO_REQUESTED          정보공개청구 중
APPROVED                승인 확정
REJECTED                불승인
CLOSED                  종결
OBJECTION               이의제기 진행 중 (이의제기 관리로 이관)
WAGE_CORRECTION         평균임금 정정 청구 (파생 사건 생성)
```

### 3-5. 소음성 난청 절차 흐름

```
CONSULTING
  └─ 초진 통과 (편측 PTA 40dB↑) + 약정서·위임장 수령 → CONTRACTED
  └─ 기본자료 수집 시작 → DOC_COLLECTING
  └─ 청구서식 + 우편 발송 (스캔 후 텔레그램 난청 공유방 공유) → SUBMITTED
  └─ 특진진찰요구서 수령 (웹팩스) → EXAM_REQUESTED
  └─ 특진병원 선택확인서 제출 → EXAM_CLINIC_SELECTED
  └─ 특진 일정 확정 → EXAM_SCHEDULED
  └─ 1차 특진 → IN_EXAM (examRound=1)
  └─ 2차 특진 → IN_EXAM (examRound=2)
  └─ 3차 특진 + 의무기록지 발급 → EXAM_DONE
  └─ 전문조사진찰요구서 수령 → EXPERT_REQUESTED
  └─ 전문조사기관 선택 → EXPERT_CLINIC_SELECTED
  └─ 전문조사 실시 → EXPERT_DONE
  └─ 통장사본 요청 수령 → BANK_REQUESTED
  └─ 통장사본 제출 → BANK_SUBMITTED
  └─ 장해등급결정통지서 수령 → DECISION_RECEIVED
  └─ 노무사 검토 → REVIEWING
      ├─ 정공 필요 → INFO_REQUESTED → 자료 수령 후 재검토
      ├─ 이상 없음 → CLOSED
      ├─ 평균임금 정정 → WAGE_CORRECTION (파생 사건)
      └─ 불승인 → REJECTED
          ├─ 종결 → CLOSED
          └─ 이의제기 → OBJECTION
```

## 5. RAG 시스템 개발 스펙 (Phase 6)

### NAS 연결 방식
```
Synology NAS (WebDAV) → Python 인덱서 → pgvector → TBSS API
```

### pgvector 스키마
```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE document_chunks (
  id           TEXT PRIMARY KEY,
  file_id      TEXT NOT NULL,
  file_name    TEXT NOT NULL,
  folder_path  TEXT NOT NULL,
  nas_path     TEXT NOT NULL,        -- NAS 내부 경로
  nas_link     TEXT,                 -- WebDAV 접근 URL (또는 공유 링크)
  chunk_text   TEXT NOT NULL,
  embedding    vector(1536),         -- OpenAI text-embedding-3-small
  doc_type     TEXT,                 -- 승불자료 / 운영자료 / 지침 / 기타
  disease      TEXT,                 -- 소음성난청 / 근골격계 / 진폐 / ...
  result       TEXT,                 -- 승인 / 불승인 / 해당없음
  indexed_at   TIMESTAMP DEFAULT NOW(),
  updated_at   TIMESTAMP DEFAULT NOW()
);

CREATE INDEX ON document_chunks USING ivfflat (embedding vector_cosine_ops);
```

### 파일 유형별 처리
| 파일 유형 | 처리 방식 |
|-----------|-----------|
| 텍스트 PDF | 텍스트 직접 추출 → 청크 분할 |
| 스캔 PDF (80% 이상) | 파일명 + 폴더 경로 메타데이터만 인덱싱 |
| Excel | 시트명 + 헤더 + 셀 데이터 추출 |
| Word (.docx) | 텍스트 직접 추출 |

## 6. 텔레그램 webhook 아키텍처 (Phase 4 재설계)

### 수신 (텔레그램 → TBSS)
```
텔레그램 메시지
  → POST /api/telegram/webhook
  → 메시지 파싱 (키워드/커맨드 감지)
  → 분기:
      /조회 [이름] → DB 조회 → Bot API sendMessage
      패턴 감지 → DB INSERT (자동 저장)
      데이터 입력 커맨드 → TBSS 입력 트리거
```

### 발신 (TBSS → 텔레그램)
```
TBSS 크론잡 (매일 오전 9시)
  → 당일 기일 도래 / To Do 마감 조회
  → Bot API sendMessage → 담당자 텔레그램 DM 또는 그룹방
```

### 자연어 조회 (Claude API 연동)
```
봇 질의 → /api/telegram/webhook
  → Claude API (자연어 파싱)
  → DB 조회
  → 결과 포맷팅 → sendMessage
```

| Railway (not Vercel) | Puppeteer Chrome 실행 + PostgreSQL 동일 서버 필요 |
| pdf-lib (not Puppeteer) | Railway 512MB 메모리 한계 내 동작 |
| 클라이언트 사이드 Excel 파싱 | 서버 사이드 xlsx 파싱 시 OOM 에러 발생 |
| BATCH_SIZE=50 JSON 배치 | Excel 임포트 대량 처리 안정성 |
| TS 상수 하드코딩 | 정적 데이터 동적 로딩 불필요한 복잡도 제거 |
| pgvector (not Pinecone) | 기존 Railway PostgreSQL 재활용, 외부 서비스 비용 절감 |
| NAS WebDAV (not 구글 드라이브) | 기존 2TB NAS 자산 직접 활용, 이관 비용 없음 |
| React Native/Expo (not PWA) | iOS/Android 네이티브 기능 접근 및 앱스토어 배포 |
| Auth.js v5 AUTH_URL 환경변수 | 미설정 시 로그아웃이 localhost로 리다이렉트되는 버그 |
