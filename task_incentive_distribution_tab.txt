## 세션 타입: 기능 개발 세션
## 작업 개요
지사장 관리·운영 페이지(/branch)에 '정산 인센티브 분배' 탭을 신규 추가한다.
기존 법인 실적 관리 탭의 '정산내역'(SettlementRecord + SettlementAllocation)을 데이터 소스로 활용하여
인센티브를 산출·분배·집계하는 기능을 구현한다.

- 새 탭 '정산 인센티브 분배'만 추가한다
- 기존 PerformanceTab.tsx, MinutesTab.tsx, RegulationsTab.tsx 등 기존 컴포넌트 파일 수정 금지

## STEP 2: API Routes 생성 (3개)

### 2-1. 인센티브 대상 정산 조회 API

파일: app/api/branch/incentive/route.ts

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { auth } from '@/auth'
import { prisma } from '@/lib/prisma'

// GET: 해당 기간·지사의 더보상 사건 정산 데이터 + 배분 내역 조회
export async function GET(req: NextRequest) {
  const session = await auth()
  if (!session?.user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  const { searchParams } = new URL(req.url)
  const branchName = searchParams.get('branchName') || ''
  const year = parseInt(searchParams.get('year') || String(new Date().getFullYear()))
  const quarter = parseInt(searchParams.get('quarter') || '1')

  const startMonth = (quarter - 1) * 3 + 1
  const months = [startMonth, startMonth + 1, startMonth + 2]

  // 더보상 TF 사건만 필터 (tfName이 '더보상'으로 시작하는 것)
  const records = await prisma.settlementRecord.findMany({
    where: {
      branchName,
      year,
      month: { in: months },
      tfName: { startsWith: '더보상' },
    },
    include: {
      allocations: true,
    },
    orderBy: [{ month: 'asc' }, { paymentDate: 'asc' }],
  })

  return NextResponse.json(records)
}
```

### 2-2. 지사 인센티브 사용 내역 API

파일: app/api/branch/incentive/usage/route.ts

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { auth } from '@/auth'
import { prisma } from '@/lib/prisma'

// GET: 해당 분기 지사 인센티브 사용 내역 조회
export async function GET(req: NextRequest) {
  const session = await auth()
  if (!session?.user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  const { searchParams } = new URL(req.url)
  const branchName = searchParams.get('branchName') || ''
  const year = parseInt(searchParams.get('year') || String(new Date().getFullYear()))
  const quarter = parseInt(searchParams.get('quarter') || '1')

  const usages = await prisma.branchIncentiveUsage.findMany({
    where: { branchName, year, quarter },
    orderBy: { usageDate: 'asc' },
  })

  return NextResponse.json(usages)
}

// POST: 지사 인센티브 사용 내역 추가
export async function POST(req: NextRequest) {
  const session = await auth()
  if (!session?.user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  const body = await req.json()
  const { branchName, year, quarter, usageDate, description, amount } = body

  if (!branchName || !year || !quarter || !usageDate || !description || !amount) {
    return NextResponse.json({ error: '필수 항목 누락' }, { status: 400 })
  }

  const usage = await prisma.branchIncentiveUsage.create({
    data: {
      branchName,
      year,
      quarter,
      usageDate: new Date(usageDate),
      description,
      amount: parseInt(amount),
    },
  })

  return NextResponse.json(usage)
}

// DELETE: 지사 인센티브 사용 내역 삭제
export async function DELETE(req: NextRequest) {
  const session = await auth()
  if (!session?.user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  const { searchParams } = new URL(req.url)
  const id = searchParams.get('id')

  if (!id) return NextResponse.json({ error: 'id 필수' }, { status: 400 })

  await prisma.branchIncentiveUsage.delete({ where: { id } })
  return NextResponse.json({ ok: true })
}
```

### 핵심 비즈니스 로직

#### 인센티브 계산 공식
```
인센티브 대상액(건별) = 정산입금액(grossAmount) ÷ 1.1 × 0.1
                     = grossAmount / 11
```
즉 부가세 제외한 금액의 10%.

#### 개인별 인센티브 = 인센티브 대상액 × (배분비율 / 100)
```
personIncentive = Math.floor(grossAmount / 11 * ratio / 100)
```

#### 지사 인센티브 적립 조건
- 담당자(salesStaffName 또는 settlementStaffName)가 '더보상'으로 시작하는 사건 = 지사 사건
- 해당 사건의 비율합계(모든 allocation의 ratio 합)가 100 미만인 경우
- 미배분율 = 100 - 비율합계
- 지사인센 적립액 = Math.floor(인센티브 대상액 × 미배분율 / 100)

#### 지사 인센 잔액 = 누적 적립액 - 사용 내역 합계

### UI 3섹션 구성

#### 섹션 1: 정산 인센티브 배분 내역 (메인 테이블)
- 기존 '정산내역' SettlementRecord 데이터 중 tfName이 '더보상'으로 시작하는 건만 표시
- 컬럼: 연번 | 정산월 | 재해자 | 사건종류 | 담당자 | 정산액(부가세포함) | 정산액(부가세제외) | 인센대상액(10%) | 비율합계 | [직원별 %·금액 열쌍] | 비고
- 직원 열은 해당 지사 소속 직원 명단(인사카드 DB)에서 로드
- 각 정산건의 allocations에서 해당 직원의 ratio를 표시, 없으면 0
- 인센 금액 = Math.floor(인센대상액 × ratio / 100)
- 비율합계 = 해당 건의 모든 allocation ratio 합산
- 비율합계가 100 미만이면 빨간색 강조
- 비율합계가 100 초과면 빨간색 강조 + 경고

#### 직원별 비율 편집 기능
- 각 정산 건의 직원 % 셀을 클릭하면 인라인 편집 가능
- 비율 입력 후 저장 시 기존 SettlementAllocation PUT/POST API 호출
- 비율 합계가 100을 초과하면 저장 차단 + 경고 메시지

주의: 기존 정산내역 탭의 SettlementAllocation을 그대로 읽고 쓰는 것.
별도의 인센티브 배분 테이블을 만들지 않는다.

기존 allocation 수정 API가 없으면 신규 생성:
app/api/branch/settlement-records/[id]/allocations/route.ts

```typescript
// PUT: 특정 SettlementRecord의 allocations 일괄 교체
export async function PUT(req: NextRequest, { params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
  const session = await auth()
  if (!session?.user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  const body = await req.json()
  const { allocations } = body  // [{ staffName, ratio, isExternal }]

  // 기존 allocation 전체 삭제 후 재생성
  await prisma.settlementAllocation.deleteMany({
    where: { settlementRecordId: id },
  })

  if (allocations && allocations.length > 0) {
    await prisma.settlementAllocation.createMany({
      data: allocations.map((a: any) => ({
        settlementRecordId: id,
        staffName: a.staffName,
        ratio: parseInt(a.ratio) || 0,
        isExternal: a.isExternal || false,
      })),
    })
  }

  // 업데이트된 전체 데이터 반환
  const updated = await prisma.settlementRecord.findUnique({
    where: { id },
    include: { allocations: true },
  })

  return NextResponse.json(updated)
}
```

#### 섹션 2: 인센티브 합계 요약 (합계 시트 재현)
- 해당 분기 전체 정산 건에 대해 직원별 인센티브 금액 합산
- 컬럼: 직원명 | 개인 인센티브 합계 | 지사 인센티브(해당 시 표시) | 자차보조금(수기입력 가능) | 합계 | 절사(만원 단위 올림/내림, Math.round(합계/10000)*10000)
- 분기평가(S/A/B/C 선택 드롭다운) | 반기평가(S/A/B/C 선택 드롭다운)
- 평가 데이터는 별도 저장 로직 필요 → 일단 UI만 표시, 저장은 추후

자차보조금은 현재 시점에서는 수기 입력 필드만 두고, DB 저장은 BranchIncentiveUsage 또는 별도 필드로 처리.
→ 현 단계에서는 자차보조금·평가 컬럼은 빈 칸으로 두고, 향후 작업 예정 표시.

절사 로직:
```typescript
const rounded = Math.floor(total / 10000) * 10000
// 또는 만원 단위 반올림: Math.round(total / 10000) * 10000
// 엑셀 데이터를 보면 4454545 → 4500000 이므로 만원 미만 올림
// 100000 단위 올림으로 보임: Math.ceil(total / 100000) * 100000
// 실제 데이터: 4454545 → 4500000 (십만원 단위 올림)
//            6053636 → 6100000 (십만원 단위 올림)
//            485454  → 500000  (십만원 단위 올림)
//            204545  → 200000  (십만원 단위 내림? 아니면 자차보조금 차감 후?)
//            1900000 → 1900000 (정확히 떨어짐)
// → 십만원 단위 올림: Math.ceil(total / 100000) * 100000
// 단, 204545 → 200000은 내림. 확인 필요. 일단 만원 단위로 반올림 적용.
```

실제 절사 로직은 지사장 재량이므로, 자동계산 후 지사장이 수정 가능하게 구현.

#### 섹션 3: 지사 인센티브 관리
- 좌측: 지사인센 미배분액 누적 표시
  - 계산: 지사 사건(salesStaffName이 '더보상'으로 시작) 중 비율합계 < 100인 건의 미배분 인센티브 합산

  - 데이터: BranchIncentiveUsage API 연동

분기 단위 조회: 1분기(1~3월), 2분기(4~6월), 3분기(7~9월), 4분기(10~12월).

기존 PerformanceTab에 이미 구현된 기간·지사 선택 UI와 동일한 패턴.
부모 페이지에서 selectedYear, selectedQuarter(또는 selectedMonth), selectedBranch를 props로 받는 구조인지 확인.
아니면 IncentiveTab 내부에서 자체 셀렉터를 구현 — 기존 탭들의 패턴을 따른다.

확인 방법:

## 주의사항

1. **인센티브 대상 필터 기준**: SettlementRecord의 tfName이 '더보상'으로 시작하는 건만 대상. 이산TF(울산동부, 울산남부 등) 사건은 제외.

2. **지사 사건 식별 기준**: salesStaffName 또는 settlementStaffName이 '더보상'으로 시작하는 건 (예: '더보상울산', '더보상부경'). 이 사건에서 비율합계가 100 미만이면 미배분분이 지사인센으로 적립.

3. **비율합계 0인 경우**: 담당자가 시스템에 없어 배분 불가(예: '김수진 항목 없어 미배정')인 경우. 비고 컬럼에 사유 표시. 지사 사건이 아닌 개인 사건에서 비율합계 0이면 지사인센에 적립되지 않음.

4. **송무·누락 건**: 정산월이 '송무', '누락' 등 비정규 값인 경우 비율합계 0으로 인센 배분 대상에서 제외될 수 있음. 테이블에는 표시하되 인센 계산에서 제외 처리 (비율합계 0이면 개인 인센 0).

5. **SettlementAllocation은 기존 정산내역 탭과 공유 데이터**. IncentiveTab에서 비율을 수정하면 정산내역 탭에도 반영됨. 이는 의도된 동작.

6. **절사 금액**: 엑셀 데이터 패턴 분석 결과 십만원 단위 올림으로 추정 (Math.ceil(total/100000)*100000). 단, 지사장 재량으로 수정 가능해야 하므로 자동계산 후 수동 편집 가능 필드로 구현.

7. **기존 PerformanceTab props 패턴 준수**: IncentiveTab이 기간·지사 선택값을 받는 방식은 기존 탭과 100% 동일해야 함. Step 0에서 확인한 패턴을 정확히 복제.

8. **Next.js 15 params 패턴**: 동적 라우트에서 params는 Promise. `const { id } = await params` 필수.

9. **Windows UTF-8**: 한국어가 포함된 파일은 PowerShell `WriteAllText` + 명시적 UTF-8 인코딩 사용.
