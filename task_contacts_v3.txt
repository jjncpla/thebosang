## Step 2: Contact 모델에 jobGrade 필드 추가

### 2-1. Prisma 스키마 수정

`prisma/schema.prisma`의 Contact 모델에 아래 필드를 추가한다:

```prisma
model Contact {
  // 기존 필드들 유지...
  jobGrade     String   @default("")  // 직군: 외근직 | 내근직 | 노무사 | 등기노무사 | 기타
}
```

스키마 수정 후:
```bash
npx prisma db push
npx prisma generate
```

### 2-2. 직급 분류 로직

title(직책) 값을 기반으로 jobGrade를 자동 분류한다.

분류 기준:
- **외근직**: 과장, 선임과장, 책임과장, 차장, 팀장
- **내근직**: 사원, 주임, 대리, 선임매니저, 책임매니저, 파트장
- **노무사**: 노무사, 수습노무사, 전임노무사, 선임노무사, 공인노무사
- **등기노무사**: 수석노무사, 지사장, 권역지사장, 센터장, 부센터장, 부대표, 대표, 전무이사, 이사
- 위 항목에 해당 없으면: **기타**

### 2-3. init-contacts API에 jobGrade 자동 분류 추가

`app/api/init-contacts/route.ts` 파일 상단에 분류 함수를 추가하고,
INSERT 쿼리에 jobGrade 컬럼을 포함시킨다.

파일 상단에 추가:
```typescript
function getJobGrade(title: string): string {
  const t = title || ''
  if (/과장|선임과장|책임과장|차장|팀장/.test(t)) return '외근직'
  if (/사원|주임|대리|선임매니저|책임매니저|파트장/.test(t)) return '내근직'
  if (/수석노무사|지사장|권역지사장|센터장|부센터장|부대표|대표|전무이사|이사/.test(t)) return '등기노무사'
  if (/노무사/.test(t)) return '노무사'
  return '기타'
}
```

Contact INSERT 쿼리의 컬럼 목록과 VALUES에 jobGrade를 추가한다:
- 컬럼: `"id","firmType","firm","branch","name","title","mobile","officePhone","email","displayOrder","createdAt","updatedAt","jobGrade"`
- VALUES의 각 row 마지막에 `,'${getJobGrade(title)}'` 추가

단, init-contacts는 SQL raw query 방식이므로 TypeScript 함수를 직접 쓸 수 없다.
대신 아래처럼 CASE 표현식을 SQL에 포함시킨다:

각 row의 jobGrade 값을 삽입 시점에 title 기반으로 미리 계산해서 문자열로 넣는다.
즉, JavaScript에서 getJobGrade(title) 함수를 호출한 결과값을 SQL row에 포함시킨다.

**구체적 구현 방법**:
init-contacts route.ts 파일 전체를 다음 구조로 재작성한다:

```typescript
import { NextResponse } from 'next/server'
import { prisma } from '@/lib/prisma'

function getJobGrade(title: string): string {
  const t = title || ''
  if (/과장|선임과장|책임과장|차장|팀장/.test(t)) return '외근직'
  if (/사원|주임|대리|선임매니저|책임매니저|파트장/.test(t)) return '내근직'
  if (/수석노무사|지사장|권역지사장|센터장|부센터장|부대표|대표|전무이사|이사/.test(t)) return '등기노무사'
  if (/노무사/.test(t)) return '노무사'
  return '기타'
}

function esc(s: string): string {
  return (s || '').replace(/'/g, "''")
}

export async function GET() {
  try {
    // 테이블 생성 (jobGrade 컬럼 포함)
    await prisma.$executeRawUnsafe(`
      CREATE TABLE IF NOT EXISTS "Contact" (
        "id" TEXT NOT NULL PRIMARY KEY,
        "firmType" TEXT NOT NULL,
        "firm" TEXT NOT NULL,
        "branch" TEXT NOT NULL,
        "name" TEXT NOT NULL,
        "title" TEXT NOT NULL DEFAULT '',
        "mobile" TEXT NOT NULL DEFAULT '',
        "officePhone" TEXT NOT NULL DEFAULT '',
        "email" TEXT NOT NULL DEFAULT '',
        "displayOrder" INTEGER NOT NULL DEFAULT 0,
        "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
        "updatedAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
        "jobGrade" TEXT NOT NULL DEFAULT ''
      )
    `)
    await prisma.$executeRawUnsafe(`
      ALTER TABLE "Contact" ADD COLUMN IF NOT EXISTS "jobGrade" TEXT NOT NULL DEFAULT ''
    `)
    await prisma.$executeRawUnsafe(`CREATE TABLE IF NOT EXISTS "IsanOffice" ("id" TEXT NOT NULL PRIMARY KEY, "name" TEXT NOT NULL, "tel" TEXT NOT NULL DEFAULT '', "fax" TEXT NOT NULL DEFAULT '', "address" TEXT NOT NULL DEFAULT '')`)
    await prisma.$executeRawUnsafe(`DELETE FROM "Contact"`)
    await prisma.$executeRawUnsafe(`DELETE FROM "IsanOffice"`)

    // 더보상 + 이산 직원 데이터를 아래에 삽입
    // (기존 init-contacts의 INSERT 구문들을 그대로 유지하되,
    //  각 row 마지막에 jobGrade 값을 추가해야 함)
    // 
    // 주의: 기존 INSERT 구문의 컬럼 목록을 찾아
    // "jobGrade" 컬럼을 추가하고,
    // 각 VALUES row 끝에 적절한 jobGrade 값을 추가한다.
    //
    // 기존 컬럼: "id","firmType","firm","branch","name","title","mobile","officePhone","email","displayOrder","createdAt","updatedAt"
    // 변경 후:  "id","firmType","firm","branch","name","title","mobile","officePhone","email","displayOrder","createdAt","updatedAt","jobGrade"

    // [기존 INSERT 쿼리들을 여기에 유지]
    // 각 row의 title 값을 파싱하여 jobGrade를 결정한다
    // 예: ('id','TBOSANG','지사','이름','과장','010-...','070-...','',0,NOW(),NOW(),'외근직')

    const c = await prisma.contact.count()
    const o = await prisma.isanOffice.count()
    return NextResponse.json({ ok: true, contacts: c, offices: o })
  } catch(e: any) {
    return NextResponse.json({ ok: false, error: e.message }, { status: 500 })
  }
}
```

**실제 구현 지침**:
기존 init-contacts/route.ts를 열어 INSERT 쿼리들을 확인한 뒤:
1. 컬럼 문자열에 `,"jobGrade"` 추가
2. 각 row의 title 부분을 파싱하여 getJobGrade 결과를 row 끝에 추가
   - row 형식: `(gen_random_uuid()::text,'TBOSANG','지사명','이름','직책','010-...','070-...','',순번,NOW(),NOW())`
   - title은 5번째 값 (index 4)
   - jobGrade를 row 끝에 추가: `,'외근직')`
3. 이 작업은 sed나 스크립트로 자동화하거나 직접 수정
