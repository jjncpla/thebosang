## 멀티 에이전트 팀 구성 (2026.03~ 적용)

### 개요
Claude Code CLI 오케스트레이션 기반 멀티 에이전트 방식으로 운영.
총괄 에이전트가 이정준의 요청을 받아 서브 에이전트들에게 Task를 위임하고 결과를 통합한다.

### 에이전트 목록

| 에이전트 | system prompt | 역할 |
|---------|--------------|------|
| 총괄 (Orchestrator) | `agents/orchestrator.md` | 요청 분해 → 서브 에이전트 위임 → 결과 통합 → 이정준에게 보고 |
| 개발 | `agents/dev.md` | 코딩, 스키마 설계, 디버깅, API 구현 |
| 디자인 | `agents/design.md` | UI/UX, 컴포넌트, 레이아웃 |
| 보안 | `agents/security.md` | 권한 분기, 인증 미들웨어, 개인정보 처리 |
| 기획 | `agents/planning.md` | PRD 작성, Phase 조정, 신규 기능 구상 |

### 기획서 구조

| 문서 | 경로 | 용도 |
|------|------|------|
| 통합기획서 | `docs/통합기획서.md` | **단일 진실의 원천 (SOT)** — 아키텍처, Phase, 권한, 메뉴 |
| 개발기획서 | `docs/기획서_개발.md` | 스키마 PRD, API 명세, 기술 결정 로그 |
| 디자인기획서 | `docs/기획서_디자인.md` | 디자인 시스템, 컬러, 페이지별 UI 스펙 |
| 보안기획서 | `docs/기획서_보안.md` | 권한 매트릭스, 인증 흐름, 환경변수 |

> **충돌 시 우선순위**: 통합기획서 > 개별기획서 > 코드

### 이정준의 역할
1. 기획 구상 및 방향 전달 (Claude.ai 총괄 에이전트에게)
2. 총괄 에이전트의 작업 분해 결과 컨펌
3. 생성된 task.txt 파일을 Claude Code CLI에 파이프
4. 결과 피드백 및 다음 작업 지시

### 작업 흐름
```
이정준 (구상/기획 전달)
  → Claude.ai 총괄 에이전트 (분해 + task.txt 생성)
  → 이정준 컨펌
  → Claude Code CLI 실행 (cat task.txt | claude --dangerously-skip-permissions)
  → 결과 확인 + 피드백
  → 반복
```

### 기획서 업데이트 원칙
- 신규 기능 추가 시 반드시 해당 기획서 업데이트
- 아키텍처 변경 시 통합기획서 먼저 업데이트
- 기획서와 코드 불일치 발견 시 기획서를 기준으로 코드 수정
