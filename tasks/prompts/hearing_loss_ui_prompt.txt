# 소음성 난청 상세 UI 재구현 — Claude Code 지시사항

## 작업 개요
`app/cases/[caseId]/page.tsx`의 `HearingLossTab` 컴포넌트와 관련 API 라우트를
새로운 스키마(HearingLossDetail + HearingLossExam)에 맞게 재구현한다.

## 절대 건드리지 말 것
- HearingLossTab 이외의 모든 컴포넌트 (BasicInfoTab, CopdTab, PneumoconiosisTab, FormTab 등)
- 헤더, 탭바, 라우팅, 로딩/에러 처리 로직
- StatusBadge, InfoCard, InfoRow, AccordionHeader, DisabledSection 등 공통 UI
- COPD, 진폐 관련 API 라우트
- inputStyle, secWrap 등 공통 스타일 상수

**저장 함수:**
```ts
const saveDetail = async () => {
  setSaving(true);
  try {
    const res = await fetch(`/api/cases/${caseId}/hearing-loss`, {
      method: "PUT",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(detail),
    });
    if (!res.ok) throw new Error("저장 실패");
    setSaveMsg("저장되었습니다");
    setTimeout(() => setSaveMsg(null), 3000);
  } catch { setSaveMsg("오류가 발생했습니다"); }
  finally { setSaving(false); }
};

const saveExam = async (exam: Partial<HearingLossExam> & { examSet: string; examRound: number }) => {
  const method = exam.id ? "PUT" : "POST";
  const url = exam.id
    ? `/api/cases/${caseId}/hearing-loss/exams/${exam.id}`
    : `/api/cases/${caseId}/hearing-loss/exams`;
  const res = await fetch(url, {
    method,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(exam),
  });
  if (!res.ok) throw new Error("검사 저장 실패");
  const updated: HearingLossExam = await res.json();
  setExams((prev) => {
    const idx = prev.findIndex((e) => e.examSet === exam.examSet && e.examRound === exam.examRound);
    if (idx >= 0) { const next = [...prev]; next[idx] = updated; return next; }
    return [...prev, updated];
  });
};
```

## 2. API 라우트 생성/수정

### 2-1. `app/api/cases/[caseId]/hearing-loss/route.ts` 수정

GET: HearingLossDetail + exams 포함해서 반환
```ts
import { prisma } from "@/lib/prisma";
import { NextResponse } from "next/server";

export async function GET(req: Request, { params }: { params: { caseId: string } }) {
  const detail = await prisma.hearingLossDetail.findUnique({
    where: { caseId: params.caseId },
    include: { exams: { orderBy: [{ examSet: "asc" }, { examRound: "asc" }] } },
  });
  return NextResponse.json(detail);
}

export async function PUT(req: Request, { params }: { params: { caseId: string } }) {
  const body = await req.json();
  // exams 필드는 별도 API로 처리하므로 제거
  const { exams, id, caseId: _caseId, createdAt, updatedAt, ...data } = body;
  const detail = await prisma.hearingLossDetail.upsert({
    where: { caseId: params.caseId },
    create: { caseId: params.caseId, ...data },
    update: data,
    include: { exams: { orderBy: [{ examSet: "asc" }, { examRound: "asc" }] } },
  });
  return NextResponse.json(detail);
}
```

### 2-2. `app/api/cases/[caseId]/hearing-loss/exams/route.ts` 신규 생성

POST: 새 exam 행 생성
```ts
import { prisma } from "@/lib/prisma";
import { NextResponse } from "next/server";

export async function POST(req: Request, { params }: { params: { caseId: string } }) {
  const body = await req.json();
  const detail = await prisma.hearingLossDetail.findUnique({ where: { caseId: params.caseId } });
  if (!detail) return NextResponse.json({ error: "HearingLossDetail not found" }, { status: 404 });
  const { id, hearingLossDetailId, createdAt, updatedAt, ...data } = body;
  const exam = await prisma.hearingLossExam.create({
    data: { hearingLossDetailId: detail.id, ...data },
  });
  return NextResponse.json(exam);
}
```

### 2-3. `app/api/cases/[caseId]/hearing-loss/exams/[examId]/route.ts` 신규 생성

PUT: exam 업데이트
```ts
import { prisma } from "@/lib/prisma";
import { NextResponse } from "next/server";

export async function PUT(req: Request, { params }: { params: { caseId: string; examId: string } }) {
  const body = await req.json();
  const { id, hearingLossDetailId, createdAt, updatedAt, ...data } = body;
  const exam = await prisma.hearingLossExam.update({
    where: { id: params.examId },
    data,
  });
  return NextResponse.json(exam);
}
```

## 주의사항

- page.tsx의 HearingLossTab만 교체. 나머지 컴포넌트 절대 수정 금지.
- 기존 `form` 패턴(Record<string, string | number | null>) 대신 타입이 명확한 `detail` 상태 사용.
- 특진 회차 데이터는 `exams` 배열로 관리. HearingLossDetail의 flat 필드에 저장하지 않음.
- prisma import 경로는 기존 파일들 참고해서 맞출 것 (`@/lib/prisma` 또는 `@/lib/db`).
- TypeScript 에러 없이 빌드 통과할 것.
