## TASK 2: 임포트 API 엔드포인트 4개 생성

### 2-1. 상담 내역 임포트
파일: `app/api/admin/import/consultation/route.ts`

- POST multipart/form-data 수신 (파일 첨부)
- xlsx로 파싱, '총 접수현황' 시트 읽기
- 행6을 헤더로 인식 (0-indexed: row index 5)
- 행7부터 데이터
- 각 행을 Consultation 모델로 upsert
  - name: 성명
  - phone: 연락처 (없으면 "-" 처리)
  - ssn: 주민번호
  - address: 주소
  - caseTypes: 사건종류 쉼표 분리하여 배열로
  - routeMain: 상담경로(대분류)
  - routeSub: 상담경로(중분류)
  - routeDetail: 상담경로(소분류)
  - visitDate: 방문일자
  - status: 사건수임 매핑
    - "약정" → "약정"
    - "종결" → "종결"
    - "연락 대기" 또는 "연락대기" → "연락대기"
    - null / 기타 → "진행중"
  - memo: 비고
  - progressNote: 기준진행경과
  - managerName을 별도 컬럼으로 저장 (User FK 매칭은 건너뜀, 이름 텍스트로만)
- upsert 기준: name + phone 조합 (동일하면 업데이트, 아니면 신규)
- 결과: { success: N, skipped: M, errors: [...] } 반환

### 2-2. 최초총현황 임포트
파일: `app/api/admin/import/objection-review/route.ts`

- POST multipart/form-data
- xlsx로 파싱, 시트명 '최초총현황' 읽기 (없으면 첫 번째 시트)
- 행2가 헤더 (0-indexed: row index 1)
- 행3부터 데이터
- 각 행을 ObjectionReview 모델로 upsert
  - approvalStatus: 승인여부
  - tfName: TF (예: "울산" → "울산TF", "울동" → "울산동부TF" 변환)
    - "울산" → "울산TF"
    - "울동" 또는 "울산동부" → "울산동부TF"
    - "울산남부" → "울산남부TF"
    - "울산북부" → "울산북부TF"
    - 그 외 → 원본값 + "TF"
  - patientName: 성명
  - caseType: 사건분류
  - decisionDate: 처분일
  - progressStatus: 사건진행여부
    - "이의제기 진행" → "이의제기 진행"
    - "송무 인계" → "송무 인계"
    - "평정청구 진행" → "평정청구 진행"
    - "검토중" → "검토중"
    - "종결" → "종결"
    - null → "검토중"
  - hasInfoDisclosure: 정공 값 있으면 true
- upsert 기준: patientName + tfName + caseType + decisionDate
- 결과: { success, skipped, errors }

- 행5부터 데이터

  - retirementDate: 퇴직일자

  - disabilityGrade: 급수

  - statWageIndustry: 업종(2)

  - statWageChangeRate: 증감률(3) (Float)
  - statWageFinal: 적용평균임금(3) (Float)
  - finalSelectedWage: 최종 채택 평균임금 (Float) - 마지막 적용평균임금
  - reviewManagerName: 검토담당자
  - reviewResult: 검토결과
  - reviewDetail: 상세쟁점
  - progressNote: 진행경과
  - claimDate: 청구일

- 결과: { success, skipped, errors }

### 2-4. 이의제기 임포트
파일: `app/api/admin/import/objection-case/route.ts`

- POST multipart/form-data
- xlsx로 파싱, 시트명 '이의제기' 읽기
- 행2가 헤더 (0-indexed: row index 1)
- 행3부터 데이터
- 심사청구/재심사청구 값 파싱 함수 별도 작성:
  ```
  parseClaimField(value):
  - null/undefined → { claimDate: null, result: null, resultDate: null }
  - "기각(23-09-14)" → { result: "기각", resultDate: new Date("2023-09-14"), claimDate: null }
  - "인용(23-09-07)" → { result: "인용", resultDate: new Date("2023-09-07"), claimDate: null }
  - "제척도과" → { result: "제척도과", resultDate: null, claimDate: null }
  - Date 객체 (날짜만) → { claimDate: Date, result: null, resultDate: null }
  - 문자열 날짜 → claimDate로 파싱 시도
  ```
- 각 행을 ObjectionCase 모델로 upsert
  - approvalStatus: 승인여부
  - tfName: TF (동일한 변환 로직)
  - patientName: 성명
  - caseType: 사건분류
  - decisionDate: 처분일(안날)
  - examClaimDate / examResult / examResultDate: 심사청구 파싱 결과
  - reExamClaimDate / reExamResult / reExamResultDate: 재심사청구 파싱 결과
  - managerId: null (담당자 이름은 memo에 텍스트로 저장)
  - memo: 담당자 이름 + 비고 합쳐서 저장 (예: "담당: 이정준 / " + 비고)
  - progressStatus: 사건진행여부
    - "진행중" → "진행중"
    - "종결" → "종결"
    - "송무인계" → "송무인계"
    - "검토중" → "검토중"
    - null → "진행중"
  - litigationHandover: progressStatus === "송무인계"
- upsert 기준: patientName + tfName + decisionDate
- 결과: { success, skipped, errors }
