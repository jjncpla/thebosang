## STEP 2. app/cases/page.tsx 수정

파일을 읽은 뒤 아래 수정사항을 적용해.

### 2-1. 진행상황 필터 드롭다운 추가

현재 필터 영역에 진행상황(status) 드롭다운이 없으면 추가해.
`CASE_STATUS_LABELS`를 import해서 드롭다운 옵션으로 사용해.
state 이름: `filterStatus` (없으면 추가, 있으면 활용)
URL 파라미터: `status`

### 2-2. TF 필터 드롭다운 추가

현재 TF 필터가 없으면 추가해.
`TF_BY_BRANCH`를 `@/lib/constants/tf`에서 import해서 전체 TF 목록 드롭다운으로 표시.
state 이름: `filterTf`
URL 파라미터: `tfName`

### 2-3. 진행상황 표시 한글화

테이블 행에서 `item.status`를 그대로 표시하는 부분을 찾아서
`CASE_STATUS_LABELS[item.status] ?? item.status` 로 교체해.

### 2-4. 상세페이지 링크 일원화

테이블 행 클릭 시 이동하는 경로를 확인해.
`/cases/[caseId]` 또는 `/patients/[patientId]` 중 어디로 이동하는지 보고,
`/patients/${item.patientId}` (또는 patient 관련 id 필드) 형태로 통일해.
단, 클릭된 사건의 caseType을 쿼리파라미터로 전달:
`/patients/${item.patientId}?tab=${item.caseType}`

item에 patientId 필드가 없으면 API 응답 타입을 확인해서 올바른 필드명 사용해.

## STEP 3. app/api/cases/route.ts (또는 cases API) 수정

파일을 읽어서 GET 핸들러에 아래 파라미터 처리가 없으면 추가해:
- `tfName` → `where.tfName = tfName`
- `status` → `where.status = status` (이미 있을 수 있음, 없으면 추가)

응답 데이터에 `patientId`가 포함되어 있는지 확인하고,
없으면 include 또는 select에 추가해.
