## 세션 8 — 처분검토 메뉴명 + 드롭다운 / 기일관리 사건분류 싱크

아래 작업을 순서대로 실행해줘. 각 단계마다 해당 파일을 직접 읽고 수정해.

---

## STEP 1. 처분검토 메뉴명 수정

### 1-1. 네비게이션 메뉴명 확인 및 수정
`components/` 또는 `app/` 하위에서 사이드바/네비게이션 컴포넌트 파일을 찾아.
(파일명 예: `Sidebar.tsx`, `Navigation.tsx`, `nav.tsx`, `layout.tsx` 등)

찾은 파일에서 "처분검토" 또는 "objection/review"와 관련된 메뉴 항목을 찾아서
메뉴 표시명을 확인해. 현재 영문이거나 다른 명칭이면 **"처분 검토"** 로 변경해.

### 1-2. 처분검토 페이지 드롭다운 수정
`app/objection/review/page.tsx` 파일을 읽어.

페이지 내 TF 또는 사건분류 관련 드롭다운이 있는지 확인해.
- TF 드롭다운이 없으면: `TF_BY_BRANCH`를 `@/lib/constants/tf`에서 import하여 TF 필터 드롭다운 추가
- 사건분류(caseType) 드롭다운이 없으면: `CASE_TYPE_LABELS`를 `@/lib/constants/case`에서 import하여 상병 필터 드롭다운 추가
- 드롭다운이 이미 있으면 현재 옵션 값과 label이 한글로 표시되는지 확인하고 수정

---

## STEP 2. 기일관리 사건분류 싱크

`app/objection/deadline/page.tsx` 파일을 읽어.

### 2-1. 사건분류 필터 확인
기일관리 페이지에 사건분류(caseType) 필터가 있는지 확인해.
없으면 `CASE_TYPE_LABELS`를 import해서 드롭다운 필터 추가.

### 2-2. 처분검토와 사건분류 싱크
기일관리의 사건분류 필터 옵션이 처분검토 페이지와 동일한 상수(`CASE_TYPE_LABELS`)를 사용하는지 확인해.
다르면 동일하게 통일해.

### 2-3. API 파라미터 확인
`app/api/objection/` 하위 route 파일들을 찾아서
`caseType` 파라미터를 받아서 where 조건에 적용하는지 확인하고, 없으면 추가해.
