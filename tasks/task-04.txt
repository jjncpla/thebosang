# TASK-04: 서식 PDF 전면 재구현 + 관리자 노무사 입력 버그 수정
# 실행: cat task-04.txt | claude --dangerously-skip-permissions

## 배경
현재 생성된 PDF는 텍스트만 나열된 상태. 아래 문제 전체 수정 필요:
1. 관리자 페이지에서 노무사 정보 입력란이 안 보이는 버그
2. 7개 서식 전부 구조(선/박스/정적텍스트) 없이 변수 텍스트만 출력됨
3. 위임장 날짜 "26년" 버그 (YY → YYYY 오류)
4. 장해급여청구서: 수령계좌/확인사항/이송비/합병증예방관리/동의서 섹션 전부 누락
5. 특진의료기관선택확인서: 안내사항 전문 누락

## STEP 1: 관리자 페이지 버그 수정

파일: app/admin/users/page.tsx

문제: task-02에서 추가한 노무사 정보 편집 UI가 화면에 안 보임
원인 파악 후 수정. 가능한 원인:
- 편집 모달이 열리지 않는 상태 버그
- 조건부 렌더링 조건 오류
- 스크롤 영역 밖에 렌더링됨
- API /api/admin/users/[id] GET이 신규 필드를 반환하지 않음

수정 방향:
- 계정 목록 각 행에 "편집" 버튼이 보이는지 확인
- 편집 버튼 클릭 시 모달/폼 오픈되는지 확인
- 노무사 정보 8개 필드(licenseNo, birthDate, branchName, officeAddress, officeTel, officeFax, department, jobTitle)가 폼에 렌더링되는지 확인
- GET /api/admin/users/[id] 에서 Prisma include로 신규 필드 포함하는지 확인
- PATCH /api/admin/users/[id] 에서 신규 필드 업데이트 처리 확인

  font: PDFFont,
  value?: string,   // 값 있으면 텍스트, 없으면 밑줄만
