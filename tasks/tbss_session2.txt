## TASK 1: schema.prisma 추가 — 이의제기 관련 모델

파일: prisma/schema.prisma 끝에 추가 (기존 내용 건드리지 말 것)

```prisma
// ─────────────────────────────────────────────
// 이의제기 기일 관리 (심사/재심사 청구 및 결과)
// ─────────────────────────────────────────────
model ObjectionCase {
  id                    String    @id @default(cuid())
  reviewId              String?   // ObjectionReview FK (최초총현황과 연결)
  review                ObjectionReview? @relation(fields: [reviewId], references: [id])
  caseId                String?   // Case FK
  tfName                String    // TF명 (예: 울산동부TF)
  patientName           String
  caseType              String    // 소음성 난청 / COPD / 근골격계 등
  approvalStatus        String    // 승인 / 불승인 / 일부승인
  decisionDate          DateTime? // 처분일(안날) — 제척기간 기산일
  // 심사청구
  examClaimDate         DateTime? // 심사청구일
  examResult            String?   // 기각 / 인용 / 취하 / null(진행중)
  examResultDate        DateTime? // 심사 결정 송달일
  // 재심사청구
  reExamClaimDate       DateTime? // 재심사청구일
  reExamResult          String?   // 기각 / 인용 / 취하 / null(진행중)
  reExamResultDate      DateTime? // 재심사 결정 송달일
  // 업무상질병판정위원회 해당 여부 (질판위 거친 경우 심사 건너뛰고 재심사부터 가능)
  isQualityReview       Boolean   @default(false)
  // 담당자
  managerId             String?
  manager               User?     @relation("ObjectionManager", fields: [managerId], references: [id])
  // 진행 상태
  progressStatus        String    // 진행중 / 종결 / 송무인계 / 검토중
  memo                  String?
  // 소송 인계 관련
  litigationHandover    Boolean   @default(false) // 송무 인계 여부
  litigationMemo        String?
  // 재처분 필요 여부 (제척 도과 시 자동 계산)
  needsReDecision       Boolean   @default(false)
  createdAt             DateTime  @default(now())
  updatedAt             DateTime  @updatedAt
}

// ─────────────────────────────────────────────
// 평균임금 데이터 검토 (정보공개청구 후 입력)
// ─────────────────────────────────────────────
model WageReviewData {
  id                    String    @id @default(cuid())
  caseId                String?   // Case FK
  tfName                String
  patientName           String
  caseType              String
  decisionDate          DateTime?
  hasInfoDisclosure     Boolean   @default(false) // 정공 여부
  retirementDate        DateTime? // 퇴직일자
  diagnosisDate         DateTime? // 산정사유발생일(진단일)
  hasNationalDisability Boolean   @default(false) // 국가장애여부
  disabilityGrade       String?   // 급수
  workerType            String?   // 일용 / 상용
  // 임금 산정 결과
  comparisonWage        String?   // 비교임금 (실임금/동종임금/특례 등)
  appliedWage           String?   // 적용임금 유형
  workplaceName         String?   // 적용사업장명
  occupation1           String?   // 직종1
  occupation1Years      String?   // 직종1 직력
  occupation2           String?   // 직종2
  occupation2Years      String?
  occupation3           String?   // 직종3
  occupation3Years      String?
  // 실제임금
  baseAvgWage           Float?    // 최초산출평균임금
  basisNote             String?   // 산정 근거
  hasCommuteCoef        Boolean?  // 통상근로계수 적용여부
  changeRate            Float?    // 증감률
  finalAvgWage          Float?    // 적용평균임금
  // 동종근로자/임금구조 임금
  statWageGender        String?   // 성별
  statWageSize          String?   // 규모
  statWageIndustry      String?   // 업종
  statWageOccupation    String?   // 직종
  statWageQuarter       String?   // 적용분기
  statWageBase          Float?    // 최초산정임금
  statWageChangeRate    Float?    // 증감률
  statWageFinal         Float?    // 적용평균임금
  // 최종 산정
  finalSelectedWage     Float?    // 최종 채택 평균임금
  // 검토 결과
  reviewManagerName     String?   // 검토 담당자 (자유텍스트)
  reviewResult          String?   // 종결 / 평정진행 등
  reviewDetail          String?   // 상세 쟁점
  progressNote          String?   // 진행경과
  claimDate             DateTime? // 평정/적사변 청구일
  decisionResultDate    DateTime? // 결정일
  additionalReview      String?   // 추가 검토
  createdAt             DateTime  @default(now())
  updatedAt             DateTime  @updatedAt
}
```

ObjectionReview 모델에 relation 추가도 필요:
```
objectionCases ObjectionCase[]
```
User 모델에도 추가:
```
objectionCases ObjectionCase[] @relation("ObjectionManager")
```

추가 후 `npx prisma db push` 실행.

#### [최초총현황] 탭

**상단 필터**
- TF 선택 버튼: 전체 / 울산TF / 울산동부TF / 울산남부TF / 울산북부TF (하드코딩, 추후 DB화)
- 사건진행여부 필터: 전체 / 검토중 / 이의제기 진행 / 송무 검토 / 송무 인계 / 평정청구 진행 / 종결
- 검색: 성명

**테이블 컬럼**
승인여부(badge) | TF | 성명 | 사건분류 | 처분일 | 제척도래일(처분일+90일 자동계산) | 사건진행여부 | 정공여부 | 관리

- 제척도래일 = 처분일 + 90일
- 제척도래일이 오늘보다 지난 건: 행 전체 빨간색 배경
- 제척도래일까지 7일 이내인 건: 행 전체 노란색 배경 + ⚠️ 아이콘

**우측 상단 요약 카드**
- 이의제기 진행 중: N건
- 재처분 필요(제척도과): N건
- 송무 인계: N건
- 평정청구 진행: N건

**행 클릭 → 수정 모달**
모달 필드:
- 승인여부: select (승인 / 불승인 / 일부승인)
- TF: select (하드코딩 목록)
- 성명 (텍스트)
- 사건분류 (텍스트)
- 처분일: 날짜 picker
- 사건진행여부: select
  - 종결 / 검토중 / 이의제기 진행 / 송무 검토 / 송무 인계 / 평정청구 진행
  - "이의제기 진행" 선택 시: [기일관리 페이지에서 확인] 링크 표시
  - "송무 인계" 선택 시: [소송인계 사건으로 이동] 링크 표시 (추후 구현)
  - "평정청구 진행" 선택 시: [평균임금 데이터 검토] 탭으로 이동 링크 표시
- 정공 여부: toggle
- 메모: textarea
- [등록] 버튼

#### [평균임금 데이터 검토] 탭

**설명 문구 표시**: "정보공개청구를 통해 수령한 평균임금 산정내역서의 내용을 직접 입력하는 탭입니다."

**필터**
- TF 선택 / 검토결과 필터 / 성명 검색

**테이블 컬럼**
TF | 성명 | 사건분류 | 처분일 | 퇴직일 | 적용임금유형 | 최종평균임금 | 검토결과 | 검토담당자 | 관리

**등록/수정 모달**: WageReviewData 모델의 전체 필드를 섹션별로 구분하여 표시
- 기본 정보 섹션: TF, 성명, 사건분류, 처분일, 퇴직일, 진단일, 국가장애여부, 급수, 일용/상용
- 직종 정보 섹션: 직종1/직력, 직종2/직력, 직종3/직력

- 동종/통계임금 섹션: 성별, 규모, 업종, 직종, 적용분기, 최초산정임금, 증감률, 적용평균임금
- 최종 산정 섹션: 최종 채택 평균임금

### 페이지 구조

#### 상단 지표 카드 (4개)
- 진행 중: N건
- 오늘 기준 제척 임박(7일 이내): N건 (노란색)
- 제척 도과 / 재처분 필요: N건 (빨간색)
- 송무 인계: N건

#### 상단 필터
- 지사 선택 → 담당 TF 자동 연동
  - 울산지사: 울산TF / 이산울산북부TF
  - 울산동부지사: 울산동부TF / 이산울산동부TF
  - 울산남부지사: 울산남부TF / 이산울산남부TF
  (위 매핑은 하드코딩, 추후 DB화)
- 진행상태 필터: 전체 / 진행중 / 종결 / 송무인계
- 검색: 성명

#### 테이블 컬럼
승인여부 | TF | 성명 | 사건분류 | 처분일 | 제척도래일 | 심사청구일 | 심사결과 | 심사송달일 | 재심사청구일 | 재심사결과 | 재심사송달일 | 담당자 | 진행상태 | 관리

**색상 규칙:**
- 행별로 가장 최근 송달일 기준 계산:
  - 송달일로부터 90일(제척기간) 이미 도과: 행 전체 빨간색
  - 송달일로부터 제척기간 7일 이내: 행 전체 노란색 + ⚠️
  - 정상: 기본 배경

**제척기간 계산 로직:**
- 심사 결과가 있으면 → 심사송달일 기준으로 90일
- 재심사 결과가 있으면 → 재심사송달일 기준으로 90일
- 아무 결과도 없으면 → 처분일 기준 90일

**업무상질병판정위원회 해당 사건:**
- isQualityReview = true인 경우: 심사청구 없이 재심사청구부터 가능
- UI에서 해당 사건은 "질판위" 뱃지 표시

#### 등록/수정 모달
- 승인여부: select
- TF: select
- 성명, 사건분류 (텍스트)
- 처분일: 날짜 picker
- 질판위 해당 여부: toggle (true면 심사청구 입력란 비활성화 가능)
- 심사청구일: 날짜 picker
- 심사결과: select (기각 / 인용 / 취하 / 진행중)
- 심사송달일: 날짜 picker (심사결과가 기각/인용인 경우만 활성화)
- 재심사청구일: 날짜 picker
- 재심사결과: select (기각 / 인용 / 취하 / 진행중)
- 재심사송달일: 날짜 picker
- 담당자: User 목록 select
- 진행상태: select (진행중 / 종결 / 송무인계 / 검토중)
- 메모: textarea

#### 재처분 필요 사건 집계 패널
페이지 우측 또는 하단에 별도 카드로 표시:

- 컬럼: TF / 성명 / 사건분류 / 처분일 / 심사결과 / 재심사결과 / 소송인계메모

- 흉추: 추간판탈출증, 압박골절
- 요추: 추간판탈출증, 협착증, 척추전방전위증, 골절
- 골반/고관절: 고관절 골절, 대퇴골두무혈성괴사, 골관절염
- 슬관절: 골관절염, 반월상연골판손상, 인공관절, 전방십자인대손상
- 족관절/발: 족저근막염, 골절, 연골손상

직업성 암:
