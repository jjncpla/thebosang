## STEP 1: progressStatus별 자동 Todo 생성 규칙 정의

| progressStatus 값 | 생성할 Todo 제목 | type |
|---|---|---|
| 이의제기 진행 | "[재해자명] 이의제기 이유서 작성 필요" | OBJECTION_DEADLINE |
| 평정청구 진행 | "[재해자명] 평균임금 정정 청구 서류 준비" | WAGE_REQUEST |
| 송무 검토 | "[재해자명] 송무 검토 필요" | GENERAL |
| 송무 인계 | "[재해자명] 송무 인계 처리 필요" | GENERAL |

※ 미검토 / 검토중 / 종결은 Todo 생성 안 함

## STEP 2: API Route에 자동 Todo 생성 로직 추가

파일: app/api/objection/review/[id]/route.ts
(이미 존재하는 PATCH 핸들러 안에 추가)

기존 PATCH 핸들러 내부에서 progressStatus가 변경될 때 아래 로직을 추가.
기존 update 로직은 절대 수정하지 말고, update 완료 후 아래 블록만 추가:

```typescript
// progressStatus 변경 시 Todo 자동 생성
const TODO_TRIGGER_STATUS: Record<string, { title: string; type: string }> = {
  '이의제기 진행': { title: '이의제기 이유서 작성 필요', type: 'OBJECTION_DEADLINE' },
  '평정청구 진행': { title: '평균임금 정정 청구 서류 준비', type: 'WAGE_REQUEST' },
  '송무 검토':    { title: '송무 검토 필요', type: 'GENERAL' },
  '송무 인계':    { title: '송무 인계 처리 필요', type: 'GENERAL' },
}

if (body.progressStatus && TODO_TRIGGER_STATUS[body.progressStatus]) {
  const trigger = TODO_TRIGGER_STATUS[body.progressStatus]

  // 해당 사건+상태 조합으로 이미 생성된 Todo가 있는지 확인 (중복 방지)
  const existing = await prisma.todo.findFirst({
    where: {
      caseId: params.id,
      title: { contains: trigger.title },
      isDone: false,
    },
  })

  if (!existing) {
    // 담당자(caseManagerId) 조회
    const objectionCase = await prisma.objectionReview.findUnique({
      where: { id: params.id },
      include: { case: { select: { caseManagerId: true, patient: { select: { name: true } } } } },
    })

    const patientName = objectionCase?.case?.patient?.name ?? '재해자'
    const assignedTo = objectionCase?.case?.caseManagerId ?? session.user.id

    await prisma.todo.create({
      data: {
        title: `[${patientName}] ${trigger.title}`,
        type: trigger.type,
        caseId: params.id,
        patientName,
        assignedTo,
        dueDate: null, // 기일은 담당자가 직접 설정
      },
    })
  }
}
```

## STEP 3: 처분검토 페이지 — 상태 변경 성공 시 UI 피드백 추가

파일: app/objection/review/page.tsx (169행 드롭다운 근처)

드롭다운 onChange 또는 저장 버튼 클릭 핸들러에서,
API PATCH 성공 후 아래 조건에 해당하면 토스트/알림 메시지 표시:

```typescript
const AUTO_TODO_STATUSES = ['이의제기 진행', '평정청구 진행', '송무 검토', '송무 인계']

// PATCH 성공 후
if (AUTO_TODO_STATUSES.includes(newStatus)) {
  // 기존 알림 방식(alert, toast 등)에 맞춰 메시지 표시
  alert(`To Do List에 할일이 자동 추가되었습니다.`)
}
```

※ 기존 프로젝트에서 사용하는 알림 방식(toast 라이브러리 등)이 있다면 그 방식 그대로 사용.
   없으면 window.alert() 사용.
