## TBSS 전체 레이아웃 및 화면 구성 구현 작업

아래 내용을 기반으로 thebosang 프로젝트의 전체 레이아웃과 주요 페이지 UI를 구현해줘.
기존 파일을 최대한 존중하되, 레이아웃 쉘(Sidebar, Header, QuickLinks)은 새로 구성해도 됨.

---

### 1. 전체 레이아웃 구조

로그인 후 모든 페이지에 공통 적용되는 레이아웃:

```
┌─────────────────────────────────────────────────────┐
│ Header (높이 54px) - 햄버거 메뉴 | 페이지 제목 | 날짜 | 로그아웃 │
├──────────┬──────────────────────────────────────────┤
│          │                                          │
│ Sidebar  │  Main Content (overflow-y: auto)         │
│ (220px   │                                          │
│ 또는     │                                          │
│ 접힘60px)│                          [LINK 버튼] →   │
│          │                                          │
└──────────┴──────────────────────────────────────────┘
```

---

### 2. Sidebar 상세 스펙

- 배경: `linear-gradient(180deg, #0f1f45, #162850)`
- 너비: 펼침 220px / 접힘 60px, `transition: width 0.2s`
- Header의 ☰ 버튼으로 토글
- 최상단: TB 로고 (32×32 파란 박스) + "TBSS" 텍스트 (펼침 시만 표시)
- 하단: 로그인한 사용자 아바타 + 이름/지사 (펼침 시만 표시)

**메뉴 목록 (순서대로):**

| id | label | icon | 비고 |
|----|-------|------|------|
| todo | To Do List | ☑️ | badge: 미완료 건수 |
| cases | 사건 관리 | 📁 | 하위 메뉴 있음 |
| ↳ cases-consult | 상담 관리 목록 | | |
| ↳ cases-list | 사건 목록 | | |
| cases-view | 사건 조회 | 🔍 | |
| objection | 이의제기 관리 | ⚖️ | 하위 메뉴 있음 |
| ↳ objection-schedule | 기일 관리 | | |
| ↳ objection-write | 이유서·의견서 작성 | | |
| tf | TF 업무 | 📡 | 하위 메뉴 있음 |
| ↳ tf-monitor | 담당TF 모니터링 | | |
| ↳ tf-notice | TF 공지 | | |
| ↳ tf-stats | TF 통계 | | |

| admin | 관리자 페이지 | ⚙️ | ADMIN 전용, 반투명 처리 |

- 활성 메뉴: `borderLeft: 3px solid #3b82f6`, 배경 `rgba(59,130,246,0.2)`, 글자 `#93c5fd`

- 접힌 상태에서 상위 메뉴 클릭 시 → 사이드바 자동 펼침 + 하위 메뉴 열림

### 3. 우측 퀵링크 패널

- 화면 우측 고정 (position: fixed, top: 50%, transform: translateY(-50%))
- 기본 상태: "LINK" 세로 버튼만 표시 (파란 버튼, width: 24px)
- 클릭 시: 좌측으로 패널 슬라이드 오픈 (width: 185px)
- 패널 내부: 북마크 폴더 형태 (클릭 시 펼쳐지는 아코디언)

**링크 그룹:**
- 정부·공단: 고용노동부(https://moel.go.kr), 근로복지공단(https://www.comwel.or.kr), 공단 토탈서비스(https://total.comwel.or.kr)
- 더보상: 노무법인 더보상(https://thebosang.kr), 법무법인 더보상, 그룹웨어(https://gw.thebosang.kr/#/)
- 법령정보: 국가법령정보센터(https://law.go.kr), 산재보상보험법, 근로기준법

### 4. 각 페이지 레이아웃 스펙

#### 4-1. To Do List (기본 홈 화면)

좌우 2단 구성:
- 좌측 (flex:1): 
  - 상단: 월별 달력 (7×5 그리드, 날짜 클릭 가능, 오늘 날짜 강조, 할일 있는 날 하단 점 표시)
  - 하단: 선택된 날짜의 할일 목록 (유형 뱃지 + 내용 + 기한)
- 우측 (width: 260px):
  - 상단: 유형별 필터 체크박스 (전체 / 정산 요청 / 이의제기 작성 요청 / 제척 임박 / 일반 업무)
  - 하단: 자유 메모 textarea

#### 4-2. 사건 목록 (/cases 또는 /cases-list)

상하 구성:
- 상단 필터 바: 성명 검색 input + 상병 select + 진행단계 select + "신규 사건" 버튼
- 하단 테이블: 성명 / 주민번호 / 상병 / 담당TF / 담당자 / 상태(컬러 뱃지) / 최근변경일
  - 행 클릭 시 사건 상세 페이지로 이동
  - 상태 컬러: 진행 중(파랑), 완료(초록), 상담 중(주황), 이의제기(빨강)

#### 4-3. 사건 상세

상단 헤더 바:
- ← 목록 버튼 | 구분선 | 재해자명 + 상병 뱃지 + 상태 뱃지 | 우측: 서식 생성 버튼
- 주민번호, 담당자, TF 요약 정보

탭 메뉴 (기본 정보 / 특진 기록 / 진행 경과 / 서식 생성):

- **기본 정보 탭**: 2열 카드 (재해자 기본정보 / 사건 정보)
- **특진 기록 탭**: 소음성 난청 기준 - 테이블 (구분/진찰일/병원/500Hz/1kHz/2kHz/3kHz/4kHz/PTA/결과)
- **진행 경과 탭**: 세로 타임라인 + 하단 경과 추가 입력 폼
- **서식 생성 탭**: 3열 카드 그리드 (요양급여신청서/장해급여청구서/간병급여청구서/휴업급여청구서/유족급여청구서/장의비청구서), 각 카드에 "PDF 생성" 버튼

### 6. 구현 방식 가이드

- Next.js App Router 기준으로 구현
- 레이아웃 쉘은 `app/layout.tsx` 또는 별도 `components/AppShell.tsx`로 분리
- 로그인 페이지 (`/login`)에는 레이아웃 쉘 미적용
- 기존 `/cases`, `/cases/[caseId]` 등 라우트 구조 유지
- 사이드바 토글 상태는 클라이언트 컴포넌트에서 useState로 관리
- 퀵링크 패널은 별도 `components/QuickLinks.tsx`로 분리
- 기존 기능 로직(DB 조회, API 라우트 등)은 건드리지 말고 UI 레이아웃만 교체/추가
