### C-9. 처분결과 저장 → 처분검토 페이지 자동 반영
- 위치: 결정 탭 > 처분결과 필드
- 트리거: 처분결과 "승인" 또는 "불승인" 선택 후 저장 시
- 동작:
  1. 현재 사건(Case) 데이터를 처분검토(ObjectionReview) 레코드로 연결/생성
  2. API: POST /api/objection/review → body: { caseId, patientId, result: "승인"|"불승인", ...매핑 필드 }
  3. 처분검토 페이지(/objection/review)의 목록에 해당 사건이 나타나야 함
  4. 처분검토 페이지 필드와 일치하는 데이터만 표시 (불일치 필드는 무시)

- "승인" 선택 저장 시 추가 동작:
  1. 평균임금 데이터 검토 탭(/objection/review - 평균임금 데이터 검토 탭)에도 자동 반영
  2. API: POST /api/objection/wage-review → body: { caseId, patientId, ...평균임금 관련 필드 }
  3. 평균임금 데이터 검토 탭 필드와 일치하는 사건 데이터만 pre-fill

- 주의: 이미 ObjectionReview 레코드가 있으면 UPDATE, 없으면 CREATE (upsert)
