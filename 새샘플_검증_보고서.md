## 2. 발견된 주요 이슈

### 이슈 #1: 이봉근 경력증명서 PDF 암호화 (즉시 해결 가능) 🟢
- **원인**: `PDFDocument.load()`가 암호화 PDF 거부
- **해결**: `ignoreEncryption: true` 옵션 추가
- **영향**: 모든 보안 PDF (관청 발급 PDF 등) 처리 가능해짐
- **우선순위**: 🔴 즉시 적용

### 이슈 #2: 헤비 케이스 (35p+) 일관성 미달 🟡
- **케이스**: 강한곤 고용보험 (페이지 50p+ 추정)
- **현상**:
  - Run 1: fetch failed
  - Run 2: 일용직 691
  - Run 3: 571 (-120)
  - Run 4: 691
  - Run 5: 451 (-240)
- **추정 원인**:
  - 청크 갯수 많음 (10+청크) → 각 청크 처리에서 일부 fail
  - JSON parse error 빈발 (`text: ```json` 시작이지만 잘림)
  - LLM이 출력 토큰 한계 도달 → JSON 잘림 → 행 누락
- **해결 후보**:
  1. **maxOutputTokens 더 강화** + JSON 잘림 복구 robust화
  2. **청크 더 작게** (5p → 3p)
  3. **재시도 로직**: parse error 시 자동 재호출
  4. **Anthropic Claude로 폴백** (헤비 케이스만)

### 이슈 #3: 강한곤 고용보험 sources.연금 8건 오추출 🟡
- **현상**: 고용보험 PDF인데 연금이 8건 추출됨 (정답: 0건)
- **추정 원인**:
  - 청크 일부에서 표를 자격이력으로 잘못 인식
  - 또는 강한곤 자료 자체에 연금 표가 섞여 있음
- **해결**: prompt에 "이 문서는 고용보험만 다룸. 연금/건보 데이터가 보이면 추출하지 말 것" 명시

### 이슈 #4: 김종식/이봉근 고용보험 1회 parse error → 큰 누락 🟡
- **현상**: 5회 중 1회 청크에서 JSON 파싱 실패
- **김종식**: 정상 4회 (94건) vs 1회 (5건만)
- **이봉근**: 정상 4회 (253-277건) vs 1회 (188건)
- **해결**: parse error 시 청크 재시도

### 이슈 #5: 작은 케이스 시간이 예상보다 큼 🟡
- 강한곤 국민연금 (3p?): 11.3s → 현재 시스템 7s 대비 +60%
- 김석광 자격득실 (5p+, 26건): 37.6s
- 이봉근 자격득실 (10p+, 46건): 47.5s
- **해결**: 페이지 수가 길수록 비례 증가 (정상). 단, 1-3p는 더 빨라야 함

## 4. 즉시 처리 (자동 적용)

### Fix #1: PDF 암호화 처리 — `ignoreEncryption: true` 추가 ✅
- 변경 파일: `components/case-common/WorkHistorySection.tsx`
- 변경: `PDFDocument.load(buffer)` → `PDFDocument.load(buffer, { ignoreEncryption: true })`
- 효과: 보안 PDF (관청 발급 등) 처리 가능

### Fix #2: JSON 파싱 강화
- 마크다운 코드블록 더 robust 제거
- 잘림 복구 알고리즘 보강

### Fix #3: 청크 처리 실패 시 자동 재시도 (1회)
