## 세션 타입: 버그픽스 세션 (UI 개선)
## 작업 개요
IncentiveTab.tsx의 메인 테이블(섹션 1)만 수정한다.
현재 직원 수만큼 컬럼이 우측으로 펼쳐지는 구조 → 배정된 직원+비율만 뱃지로 압축 표시.
섹션 2(합계 요약), 섹션 3(지사 인센 관리)은 일절 수정하지 않는다.
API, 스키마 변경 없음.

## 수술적 수정 원칙
- app/branch/_components/IncentiveTab.tsx 파일 1개만 수정
- 다른 파일 수정 금지
- 기존 계산 로직(incentiveBase, getStaffRatio, getRatioSum, getStaffIncentive, staffIncentiveTotals, branchUnallocated 등) 모두 유지
- 섹션 2, 섹션 3 코드 블록 일절 수정 금지

## STEP 2: saveAllocation 함수 교체

### 삭제: 기존 saveAllocation 함수 전체 (// 비율 편집 저장 주석부터 setEditingCell(null) 까지)

### 추가: 아래 3개 함수로 교체

```typescript
// 편집 시작
const startEditing = (record: SettlementWithAlloc) => {
  setEditingRecordId(record.id)
  setEditAllocations(
    record.allocations
      .filter(a => a.ratio > 0)
      .map(a => ({ staffName: a.staffName, ratio: a.ratio }))
  )
}

// 편집 취소
const cancelEditing = () => {
  setEditingRecordId(null)
  setEditAllocations([])
}

// 편집 저장
const saveAllocations = async (recordId: string) => {
  const totalRatio = editAllocations.reduce((sum, a) => sum + a.ratio, 0)
  if (totalRatio > 100) {
    alert(`비율 합계가 100%를 초과합니다(${totalRatio}%). 저장할 수 없습니다.`)
    return
  }

  try {
    const res = await fetch(`/api/branch/settlement-records/${recordId}/allocations`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        allocations: editAllocations.filter(a => a.ratio > 0).map(a => ({
          staffName: a.staffName,
          ratio: a.ratio,
          isExternal: false,
        })),
      }),
    })
    if (res.ok) {
      const updated = await res.json()
      setRecords(prev => prev.map(r => r.id === recordId ? { ...r, allocations: updated.allocations } : r))
    } else {
      alert('저장 실패')
    }
  } catch (e) {
    console.error('배분 저장 실패:', e)
  }
  setEditingRecordId(null)
  setEditAllocations([])
}
```

## 교체 범위 정확한 식별 방법

1. 파일에서 `<div className="overflow-x-auto">` 를 검색한다 (섹션 1 안에 있는 것)
2. 해당 div의 여는 태그부터 닫는 `</div>` 까지가 교체 대상
3. 이 블록은 `{records.length === 0 ?` 의 삼항연산자 `:` (else) 뒤에 시작하는 블록이다
4. `thead`에 `rowSpan={2}`, `colSpan={2}`, `staffList.map(s =>` 가 있는 테이블이 교체 대상

주의: 섹션 2에도 `<div className="overflow-x-auto">`가 있을 수 있다. 
섹션 1의 것만 교체한다. 구분 기준: `rowSpan={2}` 가 있는 thead가 섹션 1.

## 완료 기준 체크리스트

- [ ] 메인 테이블에 직원별 컬럼이 사라짐 (가로 스크롤 불필요)
- [ ] 컬럼 구성: 연번 | 정산월 | 재해자 | 사건종류 | 담당자 | 정산액(VAT제외) | 인센대상액(10%) | 배분 내역 | 비고
- [ ] 배정된 직원만 뱃지로 표시 (이름 + % + 금액)
- [ ] 미배정 건은 '미배정' 텍스트
- [ ] 비율합계 < 100이면 '지사인센 N%' 또는 '미배분 N%' 강조 표시
- [ ] 뱃지 영역 클릭 시 인라인 편집 모드 (직원 추가/제거/비율 수정)
- [ ] 편집 모드 합계 100% 초과 시 저장 비활성화
- [ ] 저장 후 데이터 새로고침
- [ ] 합계 행에 직원별 합계를 뱃지로 표시
- [ ] 섹션 2(합계 요약), 섹션 3(지사 인센 관리) 변화 없음
- [ ] 기존 정산내역 탭 영향 없음

## 주의사항

- 전체 파일 재작성 절대 금지. str_replace로 정확한 블록만 교체한다.
- 교체 대상은 크게 2곳: (1) 상태변수 2줄 교체 (2) saveAllocation 함수 → 3개 함수 교체 (3) 섹션 1 테이블 블록 교체.
- 섹션 2, 섹션 3 코드는 손대지 않는다.
- 기존 함수(incentiveBase, getStaffRatio, getRatioSum, getStaffIncentive)는 그대로 유지.
- staffIncentiveTotals useMemo도 그대로 유지.
- '정산액(부가세포함)' 컬럼은 제거. '정산액(VAT제외)'와 '인센대상액(10%)'만 표시.
