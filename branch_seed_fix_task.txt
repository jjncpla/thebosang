## 세션 타입: 기능 개발 세션
## 작업 개요
1. init-branch API의 시드 데이터를 Contact.branch 형식("노무법인 더보상 X지사")으로 수정
2. lib/constants/tf.ts의 TF_BY_BRANCH 키도 동일 형식으로 업데이트
3. REGION_BRANCHES도 동일 형식으로 업데이트
4. 배포 후 init-branch 호출 → 삭제 → 재배포

---

## 수술적 수정 원칙
- 기존 파일 전체 재작성 금지 — 해당 상수만 교체
- TF 배열 값(더보상울산TF 등)은 변경하지 않음 — 키(지사명)만 변경

## Step 2: lib/constants/tf.ts TF_BY_BRANCH 키 업데이트

기존 TF_BY_BRANCH 객체의 **키만** "노무법인 더보상 X지사" 형식으로 변경한다.
**TF 배열 값은 절대 변경하지 않는다.**

교체 매핑:
```

"더보상 부산중부지사"   → "노무법인 더보상 부산중부지사"
```

TF_TO_BRANCH, ALL_TF_LIST 등 파생 상수는 TF_BY_BRANCH에서 자동 생성되므로 별도 수정 불필요.

## Step 3: lib/constants/regions.ts REGION_BRANCHES 업데이트

기존 REGION_BRANCHES 객체의 **값(지사명 배열)** 을 Contact.branch 형식으로 변경.
**기존 권역명(키)은 유지하되**, 조사 결과 기반으로 정확한 권역 매핑 적용.

현재 regions.ts의 REGION_BRANCHES를 아래로 교체:

## 배포 후 조치

## 주의사항
- TF 배열 값(더보상울산TF 등)은 변경 금지 — Case.tfName 에 이미 저장된 값과 일치해야 함
- TF_BY_BRANCH 키만 변경 — 파생 상수(TF_TO_BRANCH, ALL_TF_LIST)는 자동 재계산됨
- Branch.name = Contact.branch 정확 일치 필수 — 소속 인원 쿼리 기준
