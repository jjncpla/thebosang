## 작업 개요
소음성난청 사건 직업력 섹션에 PDF 자동 분석 기능을 추가한다.
사용자가 건강보험/국민연금/고용보험/소득금액증명원 PDF를 업로드하면
Claude Vision API가 자동으로 직업력을 추출해 rawData에 채워준다.

## 수정 원칙
- 외부 패키지 신규 설치 금지
- 기존 코드 구조 최대한 유지, 외과적 수정만
- 새 파일 2개 생성 + 기존 파일 1개 수정

## 작업 1: API 엔드포인트 생성
파일 경로: `app/api/cases/[caseId]/work-history/analyze/route.ts` (신규)

```typescript
import { NextRequest, NextResponse } from "next/server"
import { auth } from "@/auth"
import { prisma } from "@/lib/prisma"

export async function POST(
  req: NextRequest,
  { params }: { params: Promise<{ caseId: string }> }
) {
  const session = await auth()
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 })

  const { caseId } = await params

  // multipart form에서 PDF 파일들 받기
  const formData = await req.formData()
  const files = formData.getAll("files") as File[]

  if (!files || files.length === 0) {
    return NextResponse.json({ error: "파일이 없습니다" }, { status: 400 })
  }

  // 각 파일을 base64로 변환
  const pdfContents: { name: string; base64: string }[] = []
  for (const file of files) {
    const buffer = await file.arrayBuffer()
    const base64 = Buffer.from(buffer).toString("base64")
    pdfContents.push({ name: file.name, base64 })
  }

  // Claude API 호출
  const apiKey = process.env.ANTHROPIC_API_KEY
  if (!apiKey) return NextResponse.json({ error: "API key missing" }, { status: 500 })

  // 메시지 구성: 각 PDF를 document로 전송
  const userContent: unknown[] = []

  for (const pdf of pdfContents) {
    userContent.push({
      type: "document",
      source: {
        type: "base64",
        media_type: "application/pdf",
        data: pdf.base64,
      },
    })
  }

  userContent.push({
    type: "text",
    text: `위 PDF 문서들은 산재보험 청구를 위한 재해자의 직업력 자료입니다.
문서 종류: 건강보험자격득실확인서, 국민연금 가입증명/가입내역확인서, 고용보험 자격이력내역서, 소득금액증명원 중 일부 또는 전부가 포함되어 있습니다.

모든 문서를 분석하여 재해자의 전체 직업력을 추출해주세요.

반드시 아래 JSON 형식으로만 응답하세요. 다른 텍스트는 절대 포함하지 마세요:

{
  "name": "재해자 성명",
  "sources": {
    "고용산재": [
      { "company": "사업장명", "startYear": 2000, "startMonth": 1, "endYear": 2005, "endMonth": 12, "department": "", "jobType": "" }
    ],
    "건보": [
      { "company": "사업장명", "startYear": 2000, "startMonth": 1, "endYear": 2005, "endMonth": 12, "department": "", "jobType": "" }
    ],
    "소득금액": [
      { "company": "사업장명", "startYear": 2000, "startMonth": 1, "endYear": 2005, "endMonth": 12, "department": "", "jobType": "" }
    ],
    "연금": [
      { "company": "사업장명", "startYear": 2000, "startMonth": 1, "endYear": 2005, "endMonth": 12, "department": "", "jobType": "" }
    ]
  }
}

규칙:
1. 각 자료 종류별로 분리해서 sources 안에 넣는다
2. 고용보험 자격이력내역서 → "고용산재" 키 사용
3. 건강보험자격득실확인서 → "건보" 키 사용
4. 소득금액증명원 → "소득금액" 키 사용
5. 국민연금 가입증명/가입내역확인서 → "연금" 키 사용
6. 직장피부양자, 지역가입자, 피보험자 이외 신분은 제외한다 (직접 취업이력만 포함)
7. 사업장명 변경이 있는 경우 가장 최신 명칭을 사용한다
8. 소득금액증명원은 귀속연도와 법인명(상호)으로 각 연도별 사업장을 추출한다. 동일 사업장이 연속되면 하나로 합친다
9. 재직 중인 경우 endYear/endMonth는 현재 연월(2026년 1월)로 표기한다
10. startYear/endYear는 정수, startMonth/endMonth는 1~12 정수
11. 해당 자료가 없는 경우 해당 키의 배열을 빈 배열로 둔다`,
  })

  const claudeRes = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": apiKey,
      "anthropic-version": "2023-06-01",
      "anthropic-beta": "pdfs-2024-09-25",
    },
    body: JSON.stringify({
      model: "claude-opus-4-6",
      max_tokens: 4096,
      messages: [{ role: "user", content: userContent }],
    }),
  })

  if (!claudeRes.ok) {
    const err = await claudeRes.text()
    console.error("Claude API error:", err)
    return NextResponse.json({ error: "AI 분석 실패" }, { status: 500 })
  }

  const claudeData = await claudeRes.json()
  const rawText = claudeData.content?.[0]?.text ?? ""

  // JSON 파싱
  let parsed: {
    name?: string
    sources: {
      고용산재?: unknown[]
      건보?: unknown[]
      소득금액?: unknown[]
      연금?: unknown[]
    }
  }
  try {
    const cleaned = rawText.replace(/```json|```/g, "").trim()
    parsed = JSON.parse(cleaned)
  } catch {
    console.error("JSON parse error:", rawText)
    return NextResponse.json({ error: "응답 파싱 실패", raw: rawText }, { status: 500 })
  }

  // DB 저장 (workHistoryRaw에 소스별 원시 결과 저장)
  const caseWithDetail = await prisma.case.findUnique({
    where: { id: caseId },
    include: { hearingLoss: true },
  })

  if (!caseWithDetail?.hearingLoss) {
    return NextResponse.json({ error: "사건을 찾을 수 없습니다" }, { status: 404 })
  }

  await prisma.hearingLossDetail.update({
    where: { caseId },
    data: {
      workHistoryRaw: parsed.sources as object,
    },
  })

  return NextResponse.json({
    success: true,
    sources: parsed.sources,
    name: parsed.name,
  })
}
```

### 수정 위치 B: 핸들러 함수 추가

```typescript
const handlePdfAnalyze = async (e: React.ChangeEvent<HTMLInputElement>) => {
  const files = e.target.files
  if (!files || files.length === 0) return

  setIsAnalyzing(true)
  setAnalyzeError(null)

  try {
    const formData = new FormData()
    Array.from(files).forEach((file) => formData.append("files", file))

    const res = await fetch(`/api/cases/${detail.caseId}/work-history/analyze`, {
      method: "POST",
      body: formData,
    })

    if (!res.ok) {
      const err = await res.json()
      throw new Error(err.error ?? "분석 실패")
    }

    const data = await res.json()

    // rawData 상태 업데이트 — AI 분석 결과를 각 소스탭에 채워넣기
    const newRaw = { ...rawData }
    const sourceMap: Record<string, string> = {
      "고용산재": "고용산재",
      "건보": "건보",
      "소득금액": "소득금액",
      "연금": "연금",
    }
    Object.entries(sourceMap).forEach(([apiKey, stateKey]) => {
      if (data.sources?.[apiKey]?.length > 0) {
        newRaw[stateKey] = data.sources[apiKey]
      }
    })
    setD("workHistoryRaw", newRaw)

  } catch (err) {
    setAnalyzeError(err instanceof Error ? err.message : "오류가 발생했습니다")
  } finally {
    setIsAnalyzing(false)
    e.target.value = ""
  }
}
```
